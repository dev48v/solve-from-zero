🧩 Flexbox Playground — flip every container & item property, watch the boxes rearrange, then copy the flex CSS

Toggle the container on the left, click a box to edit it as an item, and read the CSS that produced the layout.

Flexbox is CSS's one-dimensional layout engine: you set display:flex on a container and its direct children become flex items laid out along one line (that can optionally wrap). Every property belongs to one of two families. Container properties — flex-direction, justify-content, align-items, align-content, flex-wrap, gap — decide the axes and how the whole group is distributed. Item properties — flex-grow, flex-shrink, flex-basis, align-self, order — let a single box grow, shrink, jump the cross axis or reorder itself. The trick that trips everyone up: justify-content works on the main axis and align-* on the cross axis, and flex-direction is what decides which is which — so the same property means "horizontal" in a row and "vertical" in a column. This tool is nothing but a state object of those properties applied as inline styles to a live box, plus a serialiser that prints the CSS. No library — just the browser's own layout.

Five things that surprise people

        
1. justify-content and align-items are NOT "horizontal" and "vertical". justify-content always works on the main axis and align-items on the cross axis. Switch flex-direction to column and the two swap meaning — justify-content now moves boxes up/down. Learn the axes, not the directions. 2. align-content does nothing on a single line. It distributes lines, so you need flex-wrap:wrap and enough items to actually wrap before it has any effect — the number-one "why isn't this working?" for align-content. 3. flex-basis beats width. On a flex item, flex-basis sets the starting size along the main axis and overrides width (in a row). flex:1 is shorthand for 1 1 0 — grow, shrink, and a zero basis, which is why flex:1 items end up equal width regardless of content. 4. order is visual only. It reorders how boxes are painted but not the DOM — so keyboard and screen-reader users still traverse the source order. Great for responsive tweaks, dangerous if you use it to fix real reading order. 5. margin:auto is a flex superpower. On a flex item, an auto margin eats all free space on that side — margin-left:auto on one item pushes it (and everything after) to the far end, the cleanest way to build a nav bar.

🔨 Build it yourself

    
A flexbox playground has almost no logic — the browser does the layout. The whole tool is one state object applied as inline styles, plus a serialiser that prints it back as CSS. Everything you see rearranging is just the browser reflowing when you change a property; your job is to give it a nice set of toggles and to turn the state into a copy-pasteable rule. The only real concept you must hold in your head is which properties live on the container and which on the item, and that justify/align follow the main/cross axes set by flex-direction. There is no library.

    
    
      
  1. 
        
    1
    The state is a container object + an items array
    
        
    You store the source of truth once: one object for the six container properties, one array of item objects (five properties each), and which item is selected. The preview and the CSS are both just views of this state.
    
        
    let container = {
  flexDirection: "row",  flexWrap: "nowrap",
  justifyContent: "flex-start", alignItems: "stretch",
  alignContent: "stretch", gap: 10
};
const box = () => ({ grow:0, shrink:1, basis:"auto", alignSelf:"auto", order:0 });
let items = [ box(), box(), box(), box() ];
let selected = 0;
    
      
    2. 
      
  2. 
        
    2
    The preview IS the CSS — apply the container props
    
        
    There is no rendering engine to write — you copy each container property straight onto the flex parent's style. Change the state, re-apply, and the browser reflows. That is the entire "live preview".
    
        
    function applyContainer(){
  const f = document.getElementById("flexbox");
  f.style.display        = "flex";
  f.style.flexDirection  = container.flexDirection;
  f.style.flexWrap       = container.flexWrap;
  f.style.justifyContent = container.justifyContent;
  f.style.alignItems     = container.alignItems;
  f.style.alignContent   = container.alignContent;
  f.style.gap            = container.gap + "px";
}
    
      
    3. 
      
  3. 
        
    3
    Render the boxes — apply the item props
    
        
    Rebuild the children from the array, dropping each item's five properties onto its inline style. Wire a click to select a box so its properties become editable. The selected box just gets an outline class.
    
        
    function renderItems(){
  const f = document.getElementById("flexbox");
  f.innerHTML = "";
  items.forEach((it, i) => {
    const b = document.createElement("div");
    b.className = "flexitem" + (i === selected ? " sel" : "");
    b.style.flexGrow   = it.grow;
    b.style.flexShrink = it.shrink;
    b.style.flexBasis  = it.basis;
    b.style.alignSelf  = it.alignSelf;
    b.style.order      = it.order;
    b.textContent = i + 1;
    b.onclick = () => { selected = i; render(); };
    f.appendChild(b);
  });
}
    
      
    4. 
      
  4. 
        
    4
    Enum properties → data-driven toggle chips
    
        
    Most flex properties are a fixed list of keywords, so build each control from data: one chip per allowed value, each writing that value into state and re-rendering. One tiny function generates every keyword control in the app.
    
        
    function chipGroup(hostId, key, values){
  const host = document.getElementById(hostId);
  values.forEach(v => {
    const b = document.createElement("button");
    b.className = "chip"; b.textContent = v;
    b.dataset.key = key; b.dataset.val = v;
    b.onclick = () => { container[key] = v; render(); };
    host.appendChild(b);
  });
}
chipGroup("cc-justifyContent", "justifyContent",
  ["flex-start","flex-end","center","space-between","space-around","space-evenly"]);
    
      
    5. 
      
  5. 
        
    5
    Main axis vs cross axis — the one idea to hold
    
        
    flex-direction sets the main axis; the cross axis is perpendicular to it. justify-content always distributes along the main axis, align-items along the cross axis — so you never hard-code left / right / top / bottom, and a single flip of direction rotates the whole layout.
    
        
    // flex-direction: row     -> main = horizontal, cross = vertical
// flex-direction: column  -> main = vertical,   cross = horizontal
//
//   justify-content  ==> MAIN axis   (start / center / space-between …)
//   align-items      ==> CROSS axis  (stretch / center / baseline …)
//
// Flip the direction and the two swap meaning automatically.
    
      
    6. 
      
  6. 
        
    6
    The flex shorthand: grow · shrink · basis
    
        
    Three item properties decide sizing along the main axis. flex-basis is the starting size; then any leftover space is shared out in proportion to flex-grow, and any overflow is removed in proportion to flex-shrink. The playground drives them with three sliders.
    
        
    grow.oninput   = () => { items[selected].grow   = +grow.value;   render(); };
shrink.oninput = () => { items[selected].shrink = +shrink.value; render(); };
basis.oninput  = () => { items[selected].basis  = basis.value + "px"; render(); };
// the CSS shorthand rolls all three into one:
//   flex: 1 1 0     // grow  shrink  basis   (== flex:1, equal columns)
//   flex: 0 0 200px // fixed 200px, never grows or shrinks
    
      
    7. 
      
  7. 
        
    7
    Per-item escape hatches: align-self & order
    
        
    Two item properties override the container. align-self lets one box ignore the container's align-items; order moves a box earlier or later visually without touching the DOM (so it stays in source order for keyboards and screen readers).
    
        
    b.style.alignSelf = it.alignSelf;  // "auto" = inherit align-items
b.style.order     = it.order;      // lower renders first; default 0
// order is a VISUAL reorder only — the DOM/tab order is unchanged,
// so never use it to fix genuine reading order.
    
      
    8. 
      
  8. 
        
    8
    Wrap makes lines; align-content spaces them
    
        
    By default a flex container is a single line and items shrink to fit. Turn on flex-wrap:wrap and overflowing items flow onto new lines — and only now does align-content have anything to do: it distributes those lines across the cross axis, exactly like justify-content does for items.
    
        
    f.style.flexWrap     = "wrap";           // allow multiple lines
f.style.alignContent = "space-between";  // distribute the LINES
// with nowrap (one line) align-content is a no-op — this is the
// single most common "align-content does nothing" gotcha.
    
      
    9. 
      
  9. 
        
    9
    Serialise the state into a CSS rule — the deliverable
    
        
    The export is the whole point: print the container object as a .container rule and the selected item as an .item-N rule. It is pure string-building — the same values you applied inline, formatted as CSS.
    
        
    function toCss(){
  const c = container, it = items[selected];
  return `.container {
  display: flex;
  flex-direction: ${c.flexDirection};
  flex-wrap: ${c.flexWrap};
  justify-content: ${c.justifyContent};
  align-items: ${c.alignItems};
  align-content: ${c.alignContent};
  gap: ${c.gap}px;
}

.item-${selected + 1} {
  order: ${it.order};
  flex: ${it.grow} ${it.shrink} ${it.basis};
  align-self: ${it.alignSelf};
}`;
}
    
      
    10. 
      
  10. 
        
    10
    Copy to clipboard · presets are canned state
    
        
    Copy is one call; the render loop keeps everything in sync. And a "preset" needs no special code — it is just a state object you drop in and re-render, so centering, a nav bar and a card row are three lines of data, not three features.
    
        
    copyBtn.onclick = () => navigator.clipboard.writeText(toCss());

const PRESETS = {
  "Centering":  { container:{ justifyContent:"center", alignItems:"center" },
                  items:[ box() ] },
  "Nav bar":    { container:{ alignItems:"center" },        // item 1 gets flex-grow:1
                  items:[ box(), grow(1), box(), box() ] },
  "Card row":   { container:{ flexWrap:"wrap", gap:16 },    // each card flex:1 1 200px
                  items:[ card(), card(), card(), card(), card(), card() ] }
};
function applyPreset(p){ Object.assign(container, p.container); items = p.items; selected = 0; render(); }
