📝 Lorem Ipsum Generator — real placeholder text, by word, sentence & paragraph

Fill any layout with convincing, meaningless text.

Lorem ipsum is placeholder copy — deliberately unreadable Latin-ish text you drop into a design so you can judge shape (line length, paragraph rhythm, where headings land) without being distracted by the actual words. This tool builds it for real in your browser: it draws from a pool of ~120 classic lorem words, assembles random-length sentences with capital letters, commas and periods, groups them into paragraphs, and can start with the famous Lorem ipsum dolor sit amet… opener. Every option is seeded, so the text stays stable while you tweak formatting and only changes when you ask for more. Output as plain text, wrapped in <p>, as a <ul><li> list, or with the odd <b>/<i> mixed in. All offline, no libraries.

Why it looks like real prose (and why you must delete it before shipping)

The classic passage is scrambled Latin from Cicero’s De Finibus Bonorum et Malorum (45 BC), so it carries the letter frequencies and word lengths of genuine Western text — it reads as prose without being readable. That realism is the point: you judge layout, not wording. But lorem is scaffolding, never a finish. Ship real content as early as you can (fake copy hides the awkward long names, empty states and edge cases real content brings), and never leave lorem in production — screen readers read it aloud, search engines index the gibberish, and it embarrassingly leaks into live pages.

🔨 Build it yourself

    
A lorem generator is a two-level pipeline with no dependencies: a word pool, a seeded random number generator, and functions that grow words into sentences and sentences into paragraphs — then a separate formatting pass that wraps the plain text in whatever HTML you need. Below is the whole thing in vanilla JS, the same code powering the LOOK tab.

    
    
      
  1. 
        
    1
    Gather a word pool
    
        
    Collect ~100–200 words from the classic passage and the wider De Finibus vocabulary. The mix of short (ut, sed) and long (consectetur, perspiciatis) words is what makes generated blocks look like natural prose instead of a monotonous list. You draw from this pool at random.
    
        
    const WORDS = ["lorem","ipsum","dolor","sit","amet",
  "consectetur","adipiscing","elit","sed","do","eiusmod",
  "tempor","incididunt","labore","dolore","magna","aliqua",
  "perspiciatis","voluptatem","accusantium","laudantium",
  /* ~110 more */ ];
    
      
    2. 
      
  2. 
        
    2
    A seeded PRNG
    
        
    Math.random() can't be seeded, so its output reshuffles on every render — maddening while you tweak options. A tiny seeded generator like mulberry32 returns the exact same sequence for a given seed, so the text is stable and reproducible. Change the seed only when you deliberately want new text.
    
        
    function mulberry32(a){
  return function(){
    a |= 0; a = (a + 0x6D2B79F5) | 0;
    let t = Math.imul(a ^ a >>> 15, 1 | a);
    t = (t + Math.imul(t ^ t >>> 7, 61 | t)) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  };
}
const rng = mulberry32(12345);      // same seed = same text
const randInt = (r,min,max) => Math.floor(r()*(max-min+1))+min;
const pick    = (r,arr)     => arr[Math.floor(r()*arr.length)];
    
      
    3. 
      
  3. 
        
    3
    Build one sentence
    
        
    A sentence is a random-length run of words: pick a length, grab that many words, capitalize the first, end with a period. Those three touches turn a bag of words into something that scans as a sentence. Vary the length so paragraphs don't look mechanical.
    
        
    const cap = w => w[0].toUpperCase() + w.slice(1);

function sentence(rng, wMin, wMax){
  const n = randInt(rng, wMin, wMax);
  const words = [];
  for (let i = 0; i < n; i++) words.push(pick(rng, WORDS));
  words[0] = cap(words[0]);
  return words.join(" ") + ".";
}
    
      
    4. 
      
  4. 
        
    4
    Add commas for rhythm
    
        
    Real prose has internal pauses. Dropping one or two commas at interior positions in longer sentences gives the text a natural rhythm — and echoes the classic passage, which breaks after amet. Keep commas off the first and last word, and don't overdo it.
    
        
    function sentence(rng, wMin, wMax){
  const n = randInt(rng, wMin, wMax);
  const words = [];
  for (let i = 0; i < n; i++) words.push(pick(rng, WORDS));
  if (n >= 6){                       // one interior comma
    const pos = randInt(rng, 2, n - 2);
    words[pos] += ",";               // "...sit amet, consectetur..."
  }
  words[0] = cap(words[0]);
  return words.join(" ") + ".";
}
    
      
    5. 
      
  5. 
        
    5
    Group sentences into paragraphs
    
        
    A paragraph is several sentences joined by spaces; a document is several paragraphs. Give the paragraph a sentence-count range so some are short and some long. This words → sentences → paragraphs hierarchy is the whole grammar of the generator.
    
        
    function paragraph(rng, sMin, sMax, wMin, wMax){
  const n = randInt(rng, sMin, sMax);
  const sents = [];
  for (let i = 0; i < n; i++) sents.push(sentence(rng, wMin, wMax));
  return sents.join(" ");
}
    
      
    6. 
      
  6. 
        
    6
    The classic opener toggle
    
        
    People expect lorem to start with the famous line. When the toggle is on, force the first sentence to begin with the canonical 8-word opener (comma after amet), then continue with random words to fill the requested length.
    
        
    const OPENER = ["lorem","ipsum","dolor","sit","amet",
                "consectetur","adipiscing","elit"];

function sentence(rng, wMin, wMax, classic){
  let words = classic ? OPENER.slice() : [];
  const n = Math.max(randInt(rng, wMin, wMax), words.length);
  while (words.length < n) words.push(pick(rng, WORDS));
  if (classic) words[4] += ",";       // "...sit amet, consectetur..."
  words[0] = cap(words[0]);
  return words.join(" ") + ".";
}
    
      
    7. 
      
  7. 
        
    7
    Format: plain, <p>, <ul><li>, random <b><i>
    
        
    Generate the plain blocks first, then apply formatting as a separate pass. That keeps the invariant simple: N paragraphs always become exactly N <p> tags, and the word count is identical whichever wrapper you pick.
    
        
    function format(blocks, fmt, rng){
  if (fmt === "plain") return blocks.join("\n\n");
  if (fmt === "p")     return blocks.map(b => `<p>${b}</p>`).join("\n");
  if (fmt === "ul")    return `<ul>\n${blocks.map(b => `  <li>${b}</li>`).join("\n")}\n</ul>`;
  if (fmt === "rich")                       // sprinkle bold / italic
    return blocks.map(b => "<p>" + b.split(" ").map(w => {
      const r = rng();
      return r < 0.06 ? `<b>${w}</b>` : r < 0.12 ? `<i>${w}</i>` : w;
    }).join(" ") + "</p>").join("\n");
}
    
      
    8. 
      
  8. 
        
    8
    The whole generate() + counts
    
        
    Chain the pieces: seed a PRNG, produce blocks for the chosen unit, format, and count. Because words are joined by single spaces and tags carry no spaces, the word count is just a whitespace split of the plain blocks — and it holds no matter how you wrap the output.
    
        
    function generate(opts){
  const rng = mulberry32(opts.seed >>> 0 || 1);
  let blocks;
  if (opts.unit === "paragraphs")
    blocks = Array.from({length: opts.count},
             (_, i) => paragraph(rng, opts.sMin, opts.sMax,
                                 opts.wMin, opts.wMax));
  else if (opts.unit === "words")
    blocks = [/* exactly opts.count words, capitalized, one period */];
  // ...sentences and list-items are the same pipeline, sliced differently
  const words = blocks.join(" ").trim().split(/\s+/).length;
  return { text: format(blocks, opts.fmt, rng), words, blocks };
}
generate({ unit:"paragraphs", count:3, seed:12345, /* ... */ });
