🔲 CSS Grid Generator — a grid is a set of tracks; you place items on the lines between them

Size the columns and rows, drop items onto the grid, copy the exact grid-template CSS.

CSS Grid is two-dimensional layout you describe as a skeleton of tracks — a list of column widths and a list of row heights — and then place items onto. The magic units are the flexible fr (a fraction of leftover space), fixed px, content-driven auto, and minmax() for "at least this, at most that". Between the tracks run grid lines, numbered from 1 — and an item is positioned by naming the line it starts on and the line it ends on (grid-column: 2 / 4 spans from line 2 to line 4). Add or remove tracks, drag across cells to span an item, tune the gaps, hit a preset, and paste the generated rule straight into your CSS. No library — the browser's own grid engine lays everything out; this tool just writes the display:grid string.

Five things that surprise people

        
1. You place items on LINES, not on cells. grid-column: 2 / 4 means "start at line 2, end at line 4" — that spans the two tracks between those lines. A 3-column grid has lines 1, 2, 3, 4. Off-by-one here is the single most common grid bug. 2. fr shares what's LEFT. The browser first subtracts every fixed track and every gap, then divides the remainder among the fr tracks in proportion — so 1fr 1fr is a 50/50 split of the leftover, not of the whole width. 3. gap is only between tracks. There is no gap before the first track or after the last; padding on the container does the edges. 4. Placing beyond your tracks creates IMPLICIT tracks. Put an item on row line 5 when you only defined 3 rows and grid quietly adds auto-sized rows to reach it — handy, but a frequent "why is there extra space?" mystery. 5. span counts tracks, lines count boundaries. grid-column: 2 / span 2 and grid-column: 2 / 4 are identical; mixing the two mental models is what trips people up.

🔨 Build it yourself

    
A grid generator has one clever idea and a lot of easy plumbing. The clever idea: to make the browser preview your layout, you hand it a real display:grid container with your track strings inlined — the browser's own grid algorithm does every measurement, so you never compute a single pixel. Everything else is data: an array of column tracks, an array of row tracks, two gap numbers, and a list of items that each carry four line numbers. The one concept you must keep straight is lines vs tracks: N tracks make N+1 lines, and items are placed by line. There is no library.

    
    
      
  1. 
        
    1
    The whole state is tracks + gaps + items
    
        
    A grid is a list of column tracks, a list of row tracks, two gap sizes, and a list of items. Each track carries a unit and value; each item carries four line numbers (cs/ce = column start/end, rs/re = row start/end). Keep it all in one plain object; that object is your document.
    
        
    let grid = {
  cols: [ {unit:"fr",v1:1}, {unit:"fr",v1:1}, {unit:"fr",v1:1} ],
  rows: [ {unit:"auto"},    {unit:"fr",v1:1} ],
  colGap: 12,
  rowGap: 12,
  items: [
    { name:"A", cs:1, ce:2, rs:1, re:2 },   // top-left cell
    { name:"B", cs:2, ce:4, rs:2, re:3 }    // spans cols 2-3 on row 2
  ]
};
    
      
    2. 
      
  2. 
        
    2
    Serialise one track to its CSS unit
    
        
    Turn a single track object into the string CSS wants. There are exactly four shapes: fr (a fraction of leftover space), px (a fixed length), auto (size to content), and minmax(min, max) (a floor and a ceiling). This tiny function is the heart of the serialiser.
    
        
    function trackStr(t){
  if (t.unit === "fr")     return t.v1 + "fr";
  if (t.unit === "px")     return t.v1 + "px";
  if (t.unit === "auto")   return "auto";
  if (t.unit === "minmax") return "minmax(" + t.v1 + "px, " + t.v2 + "fr)";
}
    
      
    3. 
      
  3. 
        
    3
    Join tracks into a template string
    
        
    grid-template-columns is just each track's unit, separated by spaces. Map the array through trackStr and join — that single line is the whole column (or row) definition. Three 1fr tracks become the string "1fr 1fr 1fr".
    
        
    function templateStr(tracks){
  return tracks.map(trackStr).join(" ");
}
// grid.cols -> "1fr 1fr 1fr"
// grid.rows -> "auto 1fr"
    
      
    4. 
      
  4. 
        
    4
    The gap shorthand is row-then-column
    
        
    One number if the two gaps match, otherwise the shorthand order is gap: row-gap column-gap (rows first — the opposite of what many expect). Collapsing equal gaps to a single value keeps the output clean.
    
        
    function gapStr(g){
  return g.rowGap === g.colGap
    ? g.rowGap + "px"                       // gap: 12px
    : g.rowGap + "px " + g.colGap + "px";   // gap: 12px 16px  (row col)
}
    
      
    5. 
      
  5. 
        
    5
    An item is four line numbers
    
        
    Placement is line-based: grid-column: start / end and grid-row: start / end. The end line is exclusive, so 2 / 4 occupies the two tracks between lines 2 and 4. Serialising an item is just dropping its four numbers into that template.
    
        
    function itemRule(it){
  return "." + it.name.toLowerCase() + " { "
       + "grid-column: " + it.cs + " / " + it.ce + "; "
       + "grid-row: "    + it.rs + " / " + it.re + "; }";
}
// { name:"B", cs:2, ce:4, rs:2, re:3 }
//   -> ".b { grid-column: 2 / 4; grid-row: 2 / 3; }"
    
      
    6. 
      
  6. 
        
    6
    Assemble the whole rule — the deliverable
    
        
    Wrap the container declarations, then append one rule per item. This string is the entire export — exactly what you paste into a stylesheet. Notice there is no math here at all: you are only describing the skeleton; the browser measures it.
    
        
    function toGridCss(g){
  const container =
    ".grid {\n" +
    "  display: grid;\n" +
    "  grid-template-columns: " + templateStr(g.cols) + ";\n" +
    "  grid-template-rows: "    + templateStr(g.rows) + ";\n" +
    "  gap: " + gapStr(g) + ";\n" +
    "}";
  const items = g.items.map(itemRule).join("\n");
  return items ? container + "\n" + items : container;
}
    
      
    7. 
      
  7. 
        
    7
    Preview = a real grid container
    
        
    This is what makes the preview real: create a display:grid element and inline the same template strings you generate. The browser's grid algorithm now owns every measurement — fr distribution, minmax clamping, gap subtraction — you never compute a pixel yourself.
    
        
    function renderPreview(g){
  const box = document.getElementById("gridPreview");
  box.style.gridTemplateColumns = templateStr(g.cols);
  box.style.gridTemplateRows    = templateStr(g.rows);
  box.style.gap = gapStr(g);               // real CSS, laid out by the engine
  box.innerHTML = ghostCells(g) + itemDivs(g);
}
    
      
    8. 
      
  8. 
        
    8
    Ghost cells reveal the tracks
    
        
    To let people see the skeleton, render one faint cell per column×row, each explicitly placed on its own line pair so it never fights the real items for auto-placement. Items then sit on top with a higher z-index. The cells are the tracks; the numbers on them are the lines you place against.
    
        
    function ghostCells(g){
  let html = "";
  for (let r = 1; r <= g.rows.length; r++)
    for (let c = 1; c <= g.cols.length; c++)
      html += `<div class="gcell" data-c="${c}" data-r="${r}"
                    style="grid-column:${c}/${c+1};grid-row:${r}/${r+1}">
                 ${c}·${r}</div>`;
  return html;   // explicit placement = no auto-flow surprises
}
    
      
    9. 
      
  9. 
        
    9
    Drag across cells → min/max lines
    
        
    Dragging to span is pure arithmetic on line numbers. Remember the cell at column c sits between lines c and c+1. So from the first cell you grabbed to the one under the cursor, the start line is the smaller index and the end line is the larger index plus one. Clamp every line to 1 … tracks+1 so an item can never point off the grid.
    
        
    function applyDrag(item, startC, startR, c, r){
  item.cs = Math.min(startC, c);
  item.ce = Math.max(startC, c) + 1;      // +1: end line is past the cell
  item.rs = Math.min(startR, r);
  item.re = Math.max(startR, r) + 1;
  render();                                // re-serialise + re-preview
}
// N tracks -> lines 1 .. N+1  (clamp against this range)

    10
    Presets are just data · copy is one call
    
        
    Every classic layout — holy-grail, a card wall, a sidebar shell — is nothing but a set of tracks plus a few placed items, so a preset is an object you drop straight into state. And the export is the one toGridCss string handed to the clipboard in a single call. That is the whole tool.
    
        
    const PRESETS = {
  "sidebar": {
    cols:[{unit:"px",v1:220},{unit:"fr",v1:1}],
    rows:[{unit:"auto"},{unit:"fr",v1:1}],
    colGap:0, rowGap:0,
    items:[ {name:"A",cs:1,ce:2,rs:1,re:3},   // sidebar spans both rows
            {name:"B",cs:2,ce:3,rs:1,re:2},   // header
            {name:"C",cs:2,ce:3,rs:2,re:3} ]  // content
  }
};
