🎞️ @keyframes Animation Builder — a keyframe is just a snapshot at a percentage; the browser tweens the gaps

Add stops on the timeline, set a look for each, watch it play, copy the exact @keyframes.

A CSS @keyframes rule is not a movie — it is a short list of snapshots, each pinned to a percentage of the animation. You describe how the element should look at 0%, maybe 50%, and 100%; the browser interpolates (tweens) every frame in between. The percentages are portions of a single loop, not seconds — how long that loop actually takes is a separate duration, how many times it repeats is iteration-count, and whether it ping-pongs is direction. Drag the dots to move a stop, tune its transform / opacity / colour, hit a preset, and paste the generated rule straight into your CSS. No library, all vanilla JS + a dynamically injected <style> tag.

Five things that surprise people

        
1. The percentages are not time. 50% means "halfway through one loop", whatever the duration is — change the duration from 1.5s to 4s and every stop stays at the same percentage but the real seconds stretch. 2. The browser fills the gaps, you don't. You give three snapshots; the engine tweens hundreds of frames between them by interpolating each animatable property. That is the whole magic word: interpolation. 3. Only some properties tween. transform, opacity and color animate smoothly; things like display or font-family just jump at the stop. Animating transform/opacity is also the fast path (the compositor handles it, no layout). 4. duration, iteration-count and direction are separate knobs. They live on the animation shorthand, not inside @keyframes — the same keyframes can play once, forever, fast, slow, or ping-pong (alternate) without changing a single stop. 5. Changing a running rule needs a restart. Rewriting the @keyframes text mid-play won't update a running animation — you must remove the animation, force a reflow, then re-add it. That "none → reflow → re-add" replay trick is exactly what this tool does on every edit.

🔨 Build it yourself

    
A keyframe builder has one clever idea and a lot of easy plumbing. The clever idea: to make the browser preview your rule, you write CSS at runtime — create a <style> element, drop a freshly-serialised @keyframes string into it, and let the real CSS engine do the interpolation. The one gotcha is that a browser will not re-read a keyframes rule that is already playing, so after every edit you perform the replay trick: set animation:none, read a layout property to force a reflow, then set the animation back. Everything else is data: an array of stops, a couple of template strings, and drag-to-move on a bar. There is no library.

    
    
      
  1. 
        
    1
    The whole state is a name + a list of stops
    
        
    An animation is a named rule plus playback settings, and inside it a list of stops. Each stop is one percentage and the property values the element should have at that percentage. Keep it all in one plain object; that object is your document. The playback fields (duration, timing, iteration, direction) are stored here too but they belong to the animation shorthand, not to @keyframes.
    
        
    let anim = {
  name: "myAnim",
  duration: 1500,        // ms — one loop
  timing: "ease",        // timing-function
  iteration: "infinite", // 1 | 2 | ... | "infinite"
  direction: "normal",   // normal | reverse | alternate | alternate-reverse
  stops: [
    { pct:0,   tx:0, ty:0, scale:1,   rotate:0,   opacity:1,  bg:"#f59e0b" },
    { pct:50,  tx:0, ty:0, scale:1.4, rotate:180, opacity:.6, bg:"#3b82f6" },
    { pct:100, tx:0, ty:0, scale:1,   rotate:0,   opacity:1,  bg:"#f59e0b" }
  ]
};
    
      
    2. 
      
  2. 
        
    2
    Serialise one stop to a declaration block
    
        
    Turn a single stop into the CSS that goes inside its percentage braces. The important detail is that all the movement/size/tilt live in one transform — translate, then scale, then rotate — because a second transform line would overwrite the first. opacity and background are their own properties.
    
        
    function stopBody(s){
  return `transform: translate(${s.tx}px, ${s.ty}px) `
       + `scale(${s.scale}) rotate(${s.rotate}deg); `
       + `opacity: ${s.opacity}; background: ${s.bg};`;
}
// order matters INSIDE transform: translate → scale → rotate
// (rotate-then-translate moves along the rotated axis — usually not what you want)
    
      
    3. 
      
  3. 
        
    3
    Assemble the @keyframes rule
    
        
    Sort the stops by percentage (the CSS spec doesn't require order, but sorted output is readable and lets you find the biggest gap when adding a stop), then wrap each one as pct% { …decls… } and nest them under @keyframes name. This string is the entire deliverable — it is exactly what you paste into a stylesheet.
    
        
    function toKeyframes(){
  const rows = [...anim.stops]
    .sort((a, b) => a.pct - b.pct)
    .map(s => `  ${s.pct}% { ${stopBody(s)} }`)
    .join("\n");
  return `@keyframes ${anim.name} {\n${rows}\n}`;
}
    
      
    4. 
      
  4. 
        
    4
    The animation shorthand ties it to an element
    
        
    @keyframes only defines the motion; an element plays it via the animation property. The shorthand order that matters here is name duration timing-function iteration-count direction. Note the duration is in seconds in CSS, so divide your millisecond state by 1000.
    
        
    function toShorthand(){
  return `animation: ${anim.name} ${anim.duration/1000}s `
       + `${anim.timing} ${anim.iteration} ${anim.direction};`;
}
// e.g. "animation: myAnim 1.5s ease infinite normal;"
    
      
    5. 
      
  5. 
        
    5
    Inject a <style> tag — write CSS at runtime
    
        
    This is what makes the preview real: create one <style> element once, keep a handle to it, and overwrite its textContent with your freshly generated rule whenever the state changes. The browser's actual CSS engine now owns the interpolation — you never compute a single in-between frame yourself.
    
        
    const styleEl = document.createElement("style");
document.head.appendChild(styleEl);          // one tag, reused forever

function writeRule(){
  styleEl.textContent = toKeyframes();       // live CSS, replaced on every edit
}
    
      
    6. 
      
  6. 
        
    6
    The replay trick (force a restart)
    
        
    Here is the one real gotcha. A browser will not pick up changes to a @keyframes rule for an animation that is already running, and re-assigning the same animation string does nothing either. The fix is a three-liner: set animation to none, then read a layout property (offsetWidth) to force a synchronous reflow so the browser commits the "no animation" state, then set the animation back. That guarantees a fresh start every time you edit.
    
        
    function apply(el){
  writeRule();
  el.style.animation = "none";
  void el.offsetWidth;                       // <-- force reflow (the crux)
  el.style.animation = anim.name + " " + (anim.duration/1000) + "s "
                     + anim.timing + " " + anim.iteration + " " + anim.direction;
}
// without the offsetWidth read, the two assignments collapse into one
// and the animation never restarts.
    
      
    7. 
      
  7. 
        
    7
    Drag a marker = map cursor-x to a percentage
    
        
    The timeline is one <div> with a marker per stop positioned by left: pct%. On pointer-down find which marker you grabbed; on move, convert the cursor's x (relative to the bar) into 0–100, clamp it, and write it back onto that stop. Pointer events cover mouse and touch with one path.
    
        
    timeline.addEventListener("pointerdown", e => {
  const m = e.target.closest(".marker");
  if (m) { dragId = +m.dataset.id; select(dragId); }
});
timeline.addEventListener("pointermove", e => {
  if (dragId == null) return;
  const r = timeline.getBoundingClientRect();
  let pct = Math.round((e.clientX - r.left) / r.width * 100);
  pct = Math.max(0, Math.min(100, pct));         // clamp to the bar
  stopById(dragId).pct = pct;
  render();                                       // redraw + re-inject rule
});
addEventListener("pointerup", () => dragId = null);
    
      
    8. 
      
  8. 
        
    8
    Add a stop in the biggest gap
    
        
    A good "add" doesn't dump a stop on top of an existing one — it finds the widest gap between consecutive percentages and drops the new stop in the middle, copying the left neighbour's look so nothing visually jumps until you edit it. Removing just deletes the selected stop, keeping a floor of two so there is always something to interpolate between.
    
        
    function addStop(){
  const s = [...anim.stops].sort((a,b)=>a.pct-b.pct);
  let gap = -1, at = 50, from = s[0];
  for (let i = 0; i < s.length - 1; i++){
    const g = s[i+1].pct - s[i].pct;
    if (g > gap){ gap = g; at = Math.round((s[i].pct + s[i+1].pct)/2); from = s[i]; }
  }
  anim.stops.push({ ...from, pct: at });   // clone neighbour, new percentage
}
    
      
    9. 
      
  9. 
        
    9
    Pause without losing position
    
        
    You never stop the loop by deleting it — you flip animation-play-state between running and paused. Paused freezes the element exactly where it is and resumes from there, which is far nicer than tearing the animation down. Keep a boolean and set the property on every animated element.
    
        
    let playing = true;
playBtn.onclick = () => {
  playing = !playing;
  targets.forEach(el => el.style.animationPlayState = playing ? "running" : "paused");
  playBtn.textContent = playing ? "⏸ Pause" : "▶ Play";
};

    10
    Presets are just data · copy is one call
    
        
    Every classic effect — pulse, bounce, shake, spin — is nothing but a canned list of stops, so a preset is an object you drop straight into state. And the export is one template string (the keyframes plus a .selector { animation: … } block) handed to the clipboard in a single call. That is the whole tool.
    
        
    const PRESETS = {
  spin:  { duration:1000, timing:"linear", stops:[
            {pct:0,rotate:0}, {pct:100,rotate:360} ] },
  pulse: { duration:1200, timing:"ease-in-out", stops:[
            {pct:0,scale:1}, {pct:50,scale:1.15}, {pct:100,scale:1} ] },
  shake: { duration:600, timing:"ease-in-out", stops:[
            {pct:0,tx:0},{pct:25,tx:-10},{pct:50,tx:10},{pct:75,tx:-10},{pct:100,tx:0} ] },
  // bounce = translateY dips: 0 → -60 → 0 → -30 → 0
};

copyBtn.onclick = () => navigator.clipboard.writeText(
  toKeyframes() + "\n\n.box {\n  " + toShorthand() + "\n}"
);
