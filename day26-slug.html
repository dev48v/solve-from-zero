🔗 Slug Generator — any title → a clean, URL-safe slug, live

Turn any headline into the tidy string you see in a URL.

A slug is the human-readable tail of a web address — the café-déjà-vu part of /blog/cafe-deja-vu. It has to be lowercase, free of spaces and punctuation, and made only of characters that survive a URL untouched. This tool does the real work in your browser: it runs Unicode NFKD normalisation to peel accents off letters (so é becomes e), transliterates a few stubborn characters (ß→ss, &→and), lowercases, turns every run of non-alphanumerics into a single separator, and trims the ends. Toggle the separator, keep-case, a max length with a word-boundary cut, and a uniqueness suffix that dodges collisions against a mock list of taken slugs. Paste a whole batch of titles at once, too. All offline, no libraries.

Emoji & non-Latin scripts

NFKD can strip accents off Latin letters, but it can't turn 東京, Москва or 🚀 into ASCII — there's no “accent” to remove. This tool drops anything it can't map (so 東京 Tokyo 🚀 → tokyo). Real transliteration of CJK, Cyrillic or Arabic needs a per-script dictionary (libraries like slugify or transliteration ship one). The alternative is to allow Unicode slugs — modern browsers show /статья fine — but then you trade ASCII portability for it.

🔨 Build it yourself

    
A slugifier is a short pipeline of string transforms with no dependencies. The order matters: normalise and strip accents before you lowercase and filter, transliterate the handful of characters Unicode won't decompose, then collapse and trim. Below is the whole thing in vanilla JS — the same code powering the LOOK tab.

    
    
      
  1. 
        
    1
    Strip accents with Unicode NFKD
    
        
    The single most important trick. String.prototype.normalize("NFKD") splits an accented letter like é into its base letter e plus a separate “combining acute accent” codepoint. Those combining marks all live in the Unicode range U+0300–U+036F, so one regex deletes every accent at once — Café becomes Cafe, Déjà becomes Deja. Do this first, while the letters are still their original case.
    
        
    function stripAccents(s){
  return s.normalize("NFKD")            // é → e + ◌́
          .replace(/[̀-ͯ]/g, ""); // drop combining marks
}
// "Café Déjà Vu!" → "Cafe Deja Vu!"
    
      
    2. 
      
  2. 
        
    2
    Transliterate what NFKD can't
    
        
    Some characters aren't a base letter plus an accent, so NFKD leaves them intact: the German ß, the ligatures æ/œ, the Scandinavian ø, and symbols like &. Map them explicitly to their ASCII spellings before normalising. Keep this table small and intentional — a full multi-script transliterator is a whole library.
    
        
    const MAP = { "ß":"ss", "æ":"ae", "œ":"oe", "ø":"o",
              "&":"and", "@":"at", "%":"percent", "€":"eur" };
function translit(s){
  return s.replace(/[^-]|[&@%]/g,
                   ch => MAP[ch] ?? ch);
}
// "Straße & Œuvre" → "Strasse and OEuvre" (then NFKD/lower)
    
      
    3. 
      
  3. 
        
    3
    Lowercase
    
        
    URLs are effectively case-insensitive for humans and many servers, and mixing case invites duplicate URLs for the same page (bad for SEO and caching). Lowercase everything so Hello-World and hello-world can never be two different slugs. Offer a keep-case escape hatch, but default to lowercase.
    
        
    s = s.toLowerCase();
// "Cafe Deja Vu!" → "cafe deja vu!"
    
      
    4. 
      
  4. 
        
    4
    Every run of non-alphanumerics → one separator
    
        
    Now the only characters allowed in the slug are a–z and 0–9. Replace every maximal run of anything else — spaces, punctuation, leftover symbols, emoji — with a single separator using [^a-z0-9]+. The + is what collapses "a & b"'s three junk characters into one hyphen instead of three.
    
        
    const sep = "-";
s = s.replace(/[^a-z0-9]+/g, sep);
// "cafe deja vu!" → "cafe-deja-vu-"
// (the "+" means runs collapse to a single "-")
    
      
    5. 
      
  5. 
        
    5
    Trim leading & trailing separators
    
        
    A title that starts or ends with punctuation — like "Café Déjà Vu!" — leaves a dangling separator (cafe-deja-vu-). Strip separators off both ends so the slug never starts or ends with a hyphen. Escape the separator in the regex in case it's a character with special meaning.
    
        
    const e = sep.replace(/[.*+?^${}()|[\]\\]/g, "\\$&");
s = s.replace(new RegExp(`^${e}+|${e}+$`, "g"), "");
// "cafe-deja-vu-" → "cafe-deja-vu"
    
      
    6. 
      
  6. 
        
    6
    Optional max length, cut on a word boundary
    
        
    Long slugs are ugly and some systems cap URL length, so allow a max. But don't chop a word in half — the-quick-brownish shouldn't truncate to the-quick-brow. Slice to the limit, and only if that slice landed inside a word, back up to the last separator. If it landed exactly on a boundary, keep it.
    
        
    if (max && s.length > max){
  const full = s;
  s = s.slice(0, max);
  if (full[max] !== sep){            // we split a word
    const cut = s.lastIndexOf(sep);
    if (cut > 0) s = s.slice(0, cut);
  }
  s = s.replace(new RegExp(`${e}+$`), ""); // tidy trailing sep
}
    
      
    7. 
      
  7. 
        
    7
    Guarantee uniqueness against a taken set
    
        
    A slug is often a database key or a route, so it must be unique. Two “Hello World” posts both want hello-world. Check the candidate against the set of slugs already in use; if it's taken, append -2, then -3, until you find a free one. In real code you'd query the DB, but the logic is identical.
    
        
    function unique(base, taken){        // taken = Set of used slugs
  if (!taken.has(base)) return base;
  let n = 2;
  while (taken.has(`${base}-${n}`)) n++;
  return `${base}-${n}`;
}
// taken has "hello-world" → returns "hello-world-2"
    
      
    8. 
      
  8. 
        
    8
    The whole slugify() — and one rule for later
    
        
    Chain the passes in order and you have a complete slugifier. The last, non-code rule matters most: once a slug is published and links point at it, never silently change it. Editing the title should not rewrite a live URL — keep the old slug or add a 301 redirect, or you break every inbound link and lose the SEO you earned.
    
        
    function slugify(text, { separator="-", keepCase=false, max=0 } = {}){
  const sep = separator, e = sep.replace(/[.*+?^${}()|[\]\\]/g,"\\$&");
  let s = translit(String(text));
  s = s.normalize("NFKD").replace(/[̀-ͯ]/g, "");
  if (!keepCase) s = s.toLowerCase();
  s = s.replace(/[^a-zA-Z0-9]+/g, sep)
       .replace(new RegExp(`^${e}+|${e}+$`,"g"), "");
  if (max && s.length > max){
    const full = s; s = s.slice(0, max);
    if (full[max] !== sep){ const c = s.lastIndexOf(sep); if (c>0) s = s.slice(0,c); }
    s = s.replace(new RegExp(`${e}+$`), "");
  }
  return s;
}
slugify("Café Déjà Vu!"); // "cafe-deja-vu"
