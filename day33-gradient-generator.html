🎨 CSS Gradient Generator — a gradient is a picture you describe, not a colour you pick

Build a gradient — drag the stops, spin the angle, copy the exact CSS.

A CSS gradient is not a colour — it is a generated image the browser paints from a short recipe: a type (linear, radial or conic), a direction (an angle, or a shape and centre), and an ordered list of colour stops (a colour at a position). The browser interpolates smoothly between the stops. This tool lets you assemble that recipe by hand and shows the exact background value it produces — the same string you paste into real CSS. No library, all vanilla JS.

Five things that surprise people

        
1. A gradient is an image, not a colour. It belongs to background-image (or the background shorthand), never background-color — so it can be layered, sized and repeated like any picture. 2. The angle points the other way to trigonometry. In CSS 0deg is straight up and angles grow clockwise, so 90deg flows to the right and 180deg downward — the opposite winding to the maths atan2 you might reach for. 3. Two stops at the same position make a hard line. red 50%, blue 50% has no blend at all — that is how you draw stripes and pie slices, not a bug. 4. Order and position both matter. Stops are read left-to-right; a stop whose position is smaller than the one before it is clamped, not moved — so reordering means re-sorting by position. 5. Conic ≠ radial. Radial spreads colour outward from a centre (a glow); conic sweeps it around a centre (a pie chart or colour wheel).

🔨 Build it yourself

    
A gradient generator is a tiny bit of string assembly wrapped in a live UI. You keep a state object — the type, the direction, and an ordered array of colour stops — and one pure function turns that state into the exact CSS background value. Everything the user drags, spins or types just mutates the state and re-renders. There is no library here; the whole engine is the function below, and the browser does the actual painting.

    
    
      
  1. 
        
    1
    Model the gradient as state
    
        
    Everything the UI can change lives in one object: the type, the direction fields for each type, and the stops array where each stop is a colour plus a position percent. Keep the array in insertion order so a knob you are dragging never jumps index mid-drag — you sort a copy only when you emit CSS.
    
        
    const state = {
  type: "linear",          // "linear" | "radial" | "conic"
  angle: 90,               // linear: degrees, 0 = up, clockwise
  shape: "circle", size: "farthest-corner", posX: 50, posY: 50,  // radial
  conicAngle: 0, conicX: 50, conicY: 50,                         // conic
  stops: [ { color: "#f59e0b", pos: 0 }, { color: "#b45309", pos: 100 } ]
};
    
      
    2. 
      
  2. 
        
    2
    Sort stops by position
    
        
    CSS reads stops left-to-right, so before you build the string you sort by position. Make it a stable sort (keep insertion order for equal positions) — that is exactly what makes a hard stop (red 50%, blue 50%) render as a crisp line in the order you added the two colours.
    
        
    function sortStops(stops){
  return stops
    .map((s, i) => ({ s, i }))                         // remember original index
    .sort((a, b) => (a.s.pos - b.s.pos) || (a.i - b.i)) // stable: ties keep order
    .map(x => x.s);
}
    
      
    3. 
      
  3. 
        
    3
    Turn stops into a CSS list
    
        
    Each stop becomes "color pos%", joined with commas. That comma-separated list is the shared body of every gradient type — the only thing that changes between linear, radial and conic is the header in front of it.
    
        
    function stopsToCss(stops){
  return sortStops(stops)
    .map(s => `${s.color} ${s.pos}%`)
    .join(", ");
}
// [{color:"#f00",pos:0},{color:"#00f",pos:100}] -> "#f00 0%, #00f 100%"
    
      
    4. 
      
  4. 
        
    4
    Assemble each gradient type
    
        
    One function, three shapes of header. linear-gradient(angle, …stops), radial-gradient(shape size at x% y%, …stops), and conic-gradient(from angle at x% y%, …stops). This is the entire engine — the same function the LOOK tab and the Node test both call.
    
        
    function buildGradient(state){
  const stops = stopsToCss(state.stops);
  if (state.type === "radial")
    return `radial-gradient(${state.shape} ${state.size} at ${state.posX}% ${state.posY}%, ${stops})`;
  if (state.type === "conic")
    return `conic-gradient(from ${state.conicAngle}deg at ${state.conicX}% ${state.conicY}%, ${stops})`;
  return `linear-gradient(${state.angle}deg, ${stops})`;   // linear
}
    
      
    5. 
      
  5. 
        
    5
    Paint the preview & output
    
        
    Apply the value to background (or background-image) on the preview element, and drop the same string into the copy box. One render() function does both, called after every state change — the browser handles the actual interpolation.
    
        
    function render(){
  const css = buildGradient(state);
  preview.style.background = css;
  cssOut.textContent = "background: " + css + ";";
}
    
      
    6. 
      
  6. 
        
    6
    Drag a stop along the track
    
        
    Position a knob with left: pos%. On pointer-move, convert the cursor's X into a 0–100 percent of the track width, clamp it, and write it back to that stop. Because the state array is unsorted, the dragged index stays put; the sort happens only when you build the CSS.
    
        
    function posFromEvent(e, track){
  const r = track.getBoundingClientRect();
  const pct = (e.clientX - r.left) / r.width * 100;
  return Math.max(0, Math.min(100, Math.round(pct)));   // clamp 0..100
}
track.addEventListener("pointermove", e => {
  if (dragIndex == null) return;
  state.stops[dragIndex].pos = posFromEvent(e, track);
  render();
});
    
      
    7. 
      
  7. 
        
    7
    The angle dial (0 = up, clockwise)
    
        
    A click on the dial gives you an offset from centre. CSS angles start at the top and grow clockwise, which is not what atan2(y, x) gives — so use atan2(dx, -dy) to get the CSS convention directly, then rotate the needle by that same value.
    
        
    function angleFromEvent(e, dial){
  const r = dial.getBoundingClientRect();
  const dx = e.clientX - (r.left + r.width / 2);
  const dy = e.clientY - (r.top + r.height / 2);
  let deg = Math.atan2(dx, -dy) * 180 / Math.PI;   // 0 = up, clockwise
  return Math.round((deg + 360) % 360);
}
needle.style.transform = `translate(-50%,-100%) rotate(${state.angle}deg)`;
    
      
    8. 
      
  8. 
        
    8
    Add, remove, reverse, randomize
    
        
    Every extra feature is just an array edit followed by render(). Add a stop where the user clicks; remove on double-click (but keep at least two); reverse by flipping each position to 100 - pos; randomize by generating a few stops with random colours and sorted positions.
    
        
    function addStop(pos, color){ state.stops.push({ pos, color }); render(); }
function removeStop(i){ if (state.stops.length > 2){ state.stops.splice(i,1); render(); } }
function reverse(){ state.stops.forEach(s => s.pos = 100 - s.pos); render(); }
function randomColor(){ return "#" + Math.floor(Math.random()*0xffffff).toString(16).padStart(6,"0"); }
