🕰️ Unix Timestamp Converter — Epoch ⇄ Human Time

Paste any epoch. It auto-detects seconds vs milliseconds and shows every format.

A Unix timestamp is just a count of seconds since 1 Jan 1970 UTC — the "epoch". This tool takes a number and figures out whether it's seconds, milliseconds, or microseconds (by its size), then renders it as UTC, your local time, ISO-8601, RFC-2822, and a relative "… ago". Or go the other way: pick a date and get the epoch back. Everything is live and offline.

🔨 Build it yourself

    
Detect the unit from the number's magnitude, build a Date, then format it every way with the built-in Intl and Date APIs. ~60 lines of vanilla JS.

    Detect seconds vs milliseconds
    
        
    JavaScript's Date works in milliseconds, but most APIs hand you seconds. Guess the unit from the digit count: a seconds timestamp for "now" is ~10 digits, milliseconds ~13, microseconds ~16. Normalise everything to milliseconds before building a Date.
    
        
    function toMillis(n){
  const digits = String(Math.trunc(Math.abs(n))).length;
  if (digits <= 11) return { ms: n * 1000,       unit: "seconds" };
  if (digits <= 14) return { ms: n,              unit: "milliseconds" };
  return               { ms: Math.round(n/1000), unit: "microseconds" };
}

    Build a Date & format UTC + ISO
    
        
    A Date is an absolute instant. toUTCString() gives the RFC-style UTC text, toISOString() the ISO-8601 form (always UTC, ending in Z). These are timezone-independent — the same instant, just written differently.
    
        
    const d = new Date(ms);
const iso = d.toISOString();      // 2025-07-01T12:00:00.000Z
const utc = d.toUTCString();      // Tue, 01 Jul 2025 12:00:00 GMT
if (isNaN(d)) return "invalid timestamp";

    Local time & timezone name via Intl
    
        
    The same instant shown in the user's zone. Intl.DateTimeFormat formats it and also reports the resolved IANA zone (e.g. Asia/Kolkata). Timezones are purely a display concern — the underlying instant never changes.
    
        
    const tz = Intl.DateTimeFormat().resolvedOptions().timeZone; // "Asia/Kolkata"
const local = d.toLocaleString(undefined, {
  dateStyle: "medium", timeStyle: "long"
});

    4
    Relative time ("3 days ago")
    
        
    Intl.RelativeTimeFormat turns a difference into human words. Compute the delta from now, pick the biggest sensible unit (years → months → … → seconds), and let Intl phrase it — localised, and correctly past or future.
    
        
    function relative(ms){
  const diff = ms - Date.now();               // +future / −past
  const rtf = new Intl.RelativeTimeFormat(undefined,{numeric:"auto"});
  const U = [["year",31536e6],["day",864e5],["hour",36e5],["minute",6e4],["second",1e3]];
  for (const [unit, size] of U)
    if (Math.abs(diff) >= size || unit === "second")
      return rtf.format(Math.round(diff/size), unit);
}

    Reverse: date → epoch
    
        
    The other direction: a datetime-local input is parsed as local time; getTime() returns milliseconds, so divide by 1000 (and floor) for seconds. This is the value you'd store in a database or send to an API.
    
        
    const d = new Date(inputEl.value);   // parsed in local time
const ms  = d.getTime();
const sec = Math.floor(ms / 1000);   // ← what most APIs expect
