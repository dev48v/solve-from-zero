📄 XML Formatter — messy XML is still a tree; tokenize it, track the depth, re-indent

Paste messy or minified XML — get it back clean, correctly nested, and validated.

XML looks like a wall of angle-brackets, but it is really a tree written as flat text: every <tag> opens a branch and every </tag> closes it. This tool doesn't lean on the browser's parser to lay it out — it runs a tiny tokenizer that splits the text into tags, text, comments and CDATA, then walks those tokens while tracking the current depth with a stack. Depth × indent gives the indentation; the same stack catches any mismatched or unclosed tag and tells you exactly which one broke. Flip to Minify and the identical token list is re-joined with the whitespace stripped. All vanilla JS, nothing uploaded.

Five things that surprise people

        
1. A raw < always starts a tag. That is why real text content has to escape it as &lt; (and & as &amp;) — a stray < in your data is not content, it is a broken tag, and every XML parser will choke on it. 2. Every tag must close — there are no void elements. In HTML <img> and <br> stand alone, but XML has no such list: you must write <img/> or <img></img>, and this validator flags a lone <img> as unclosed. 3. Self-closing and empty are the same thing. <br/> and <br></br> mean an identical empty element; the formatter picks one clean form. 4. Attribute spacing is cosmetic, values are not. Order and the gaps around = carry no meaning, so we tidy them — but the value between the quotes is preserved byte-for-byte, quotes and all. 5. Whitespace inside text can be significant. A generic re-indent could change meaning, so this tool only trims the whitespace-only gaps between element tags and never rewrites the text inside a leaf like <name>John</name>.

🔨 Build it yourself

    
A pretty-printer is not magic and it is not a regex — it is a two-pass pipeline. Pass one turns the flat text into a list of tokens (tags, text, comments). Pass two walks those tokens keeping a running depth on a stack: an open tag pushes and indents deeper, a close tag pops and outdents, and if a close tag does not match the top of the stack you have found the exact broken tag. There is no library here; the whole engine is the handful of functions below, and the same functions power both the Format and Minify buttons.

    
    
      
  1. 
        
    1
    Model XML as a flat token stream
    
        
    Before you can indent anything you need to know what each piece is. The formatter's first job is to turn the raw string into an ordered array of tokens, each tagged with a kind: an open tag, a close tag, a self-closing tag, plain text, or a comment/cdata/decl. Nesting is not stored yet — that is rebuilt in pass two — so this stays a simple left-to-right scan.
    
        
    // one token = one piece of the document, in source order
// { kind:"open"|"close"|"self"|"text"|"comment"|"cdata"|"decl", name, attrStr, raw, pos }
// e.g.  <book id="bk1">Hi</book>  ->
//   { kind:"open",  name:"book", attrStr:' id="bk1"' }
//   { kind:"text",  raw:"Hi" }
//   { kind:"close", name:"book" }
    
      
    2. 
      
  2. 
        
    2
    Scan the string, splitting at every <
    
        
    Walk a cursor through the text. If the current char is < you are at the start of some markup; otherwise you are in text, which runs until the next <. Everything hangs off that one branch. Whitespace-only text between tags is kept as a token now and dropped later, so the pass stays dumb and total.
    
        
    while (i < n){
  if (xml[i] === "<"){
    // …decide which kind of markup this is (step 3)…
  } else {
    const nx = xml.indexOf("<", i);       // text runs up to the next tag
    const end = nx === -1 ? n : nx;
    toks.push({ kind:"text", raw: xml.slice(i, end), pos: i });
    i = end;
  }
}
    
      
    3. 
      
  3. 
        
    3
    Classify the markup by its prefix
    
        
    Not everything in angle brackets is an element. Check the characters right after <: <!-- is a comment, <![CDATA[ is raw character data, <? is a processing instruction / declaration, <! is a doctype. Each of those runs to its own terminator, and if that terminator is missing you already have a clean error to report. Anything else is a real tag.
    
        
    if (xml.startsWith("<!--", i)){
  const e = xml.indexOf("-->", i);
  if (e === -1) throw new XmlError("Unclosed comment (no matching -->)", "<!--", i);
  toks.push({ kind:"comment", raw: xml.slice(i, e + 3), pos: i }); i = e + 3;
}
else if (xml.startsWith("<![CDATA[", i)){ /* run to ]]> */ }
else if (xml.startsWith("<?", i))        { /* run to ?>  (declaration) */ }
else if (xml.startsWith("<!", i))        { /* run to >   (doctype)     */ }
else { /* a real element tag — step 4 */ }
    
      
    4. 
      
  4. 
        
    4
    Find the tag's end — but respect quotes
    
        
    The naive move is indexOf(">"), but that breaks the instant an attribute value contains a >, e.g. <rule expr="a > b"/>. So scan for the closing > while tracking whether you are inside a " or ' string, and ignore any > found inside quotes. This tiny state machine is the difference between a toy and a formatter that survives real data.
    
        
    function findTagEnd(s, start){
  let quote = null;
  for (let j = start + 1; j < s.length; j++){
    const c = s[j];
    if (quote){ if (c === quote) quote = null; }   // leaving a quoted value
    else if (c === '"' || c === "'") quote = c;    // entering one
    else if (c === ">") return j;                  // real tag end
  }
  return -1;                                        // unterminated tag
}
    
      
    5. 
      
  5. 
        
    5
    Split name from attributes; detect close & self-close
    
        
    With the tag text in hand, a leading / means a close tag and a trailing / means self-closing. The name is the first run of non-space characters; whatever follows is the attribute string, kept verbatim so values are never corrupted. When you re-emit the tag you tidy the spacing around attributes but copy each value — quotes included — exactly as written.
    
        
    const isClose = raw[1] === "/";
let inner = raw.slice(isClose ? 2 : 1, -1).trim();   // drop < / and >
let self = false;
if (!isClose && inner.endsWith("/")){ self = true; inner = inner.slice(0, -1).trim(); }
const name = inner.match(/^([^\s\/>]+)/)[1];
const attrStr = inner.slice(name.length);            // preserved as-is

// re-emit: normalise spacing, keep every value byte-for-byte
function serializeTag(name, attrStr, selfClose){
  let s = "<" + name;
  for (const a of parseAttrs(attrStr)) s += " " + a.name + (a.value !== undefined ? "=" + a.value : "");
  return s + (selfClose ? "/>" : ">");
}
    
      
    6. 
      
  6. 
        
    6
    Track depth with a stack — and validate
    
        
    This is the heart of it. Keep a stack of open tags and an integer depth. An open tag prints at the current depth, pushes its name, and increments depth. A close tag must match the name on top of the stack: if the stack is empty it is a stray close, if the names differ it is a mismatch — either way you know the exact offending tag. Pop, decrement, print. Anything left on the stack at the end is an unclosed tag.
    
        
    if (t.kind === "open"){ lines.push(pad + openTag); stack.push(t); depth++; }

if (t.kind === "close"){
  if (!stack.length)
    throw new XmlError("Unexpected </" + t.name + "> — nothing is open", "</" + t.name + ">", t.pos);
  const open = stack[stack.length - 1];
  if (open.name !== t.name)
    throw new XmlError("Mismatched: <" + open.name + "> open, found </" + t.name + ">", "</" + t.name + ">", t.pos);
  stack.pop(); depth = Math.max(0, depth - 1);
  lines.push(unit.repeat(depth) + "</" + t.name + ">");
}
// after the loop: if (stack.length) -> "Unclosed tag <" + stack.pop().name + ">"
    
      
    7. 
      
  7. 
        
    7
    Indentation = depth × the indent unit
    
        
    Once depth is tracked, indenting is trivial: the prefix for a line is the indent unit (two spaces, four spaces, or a tab) repeated depth times. Every token type — text, comments, self-closing tags — is printed at the current depth; only open and close tags move it. That single rule is what turns a one-line blob into a readable tree.
    
        
    const unit = opt.indent != null ? opt.indent : "  ";   // "  " | "    " | "\t"
const pad  = unit.repeat(depth);                       // e.g. depth 3 -> 6 spaces

lines.push(pad + serializeTag(t.name, t.attrStr, true)); // self-closing at this depth
// comments / cdata / declarations print at `pad` too and never change depth
    
      
    8. 
      
  8. 
        
    8
    Keep leaf elements on one line
    
        
    Blindly putting every token on its own line makes <name>John</name> sprawl across three lines, which reads worse. So when an open tag is followed by a single text token and then its matching close, collapse all three onto one line — a two-token lookahead. Empty elements (<tag></tag>) collapse the same way.
    
        
    const a = toks[k + 1], b = toks[k + 2];
// <tag>text</tag>  ->  one line
if (a && a.kind === "text" && b && b.kind === "close" && b.name === t.name){
  lines.push(pad + openTag + a.raw.trim() + "</" + t.name + ">"); k += 2; continue;
}
// <tag></tag>  ->  one line
if (a && a.kind === "close" && a.name === t.name){
  lines.push(pad + openTag + "</" + t.name + ">"); k += 1; continue;
}
    
      
    9. 
      
  9. 
        
    9
    Minify: the same tokens, whitespace stripped
    
        
    Minify is not a separate parser — it reuses the exact same token list. Instead of adding indentation and newlines, you concatenate every tag back-to-back and trim the whitespace-only text between them (text inside a leaf is kept). That is why Format and Minify are perfect inverses of structure: one adds layout, the other removes it, both from one tokenize().
    
        
    function minifyXml(xml){
  const toks = tokenize(xml); let out = "";
  for (const t of toks){
    if (t.kind === "text"){ const x = t.raw.trim(); if (x) out += x; }
    else if (t.kind === "open")  out += serializeTag(t.name, t.attrStr, false);
    else if (t.kind === "self")  out += serializeTag(t.name, t.attrStr, true);
    else if (t.kind === "close") out += "</" + t.name + ">";
    else out += t.raw.trim();                 // comment / cdata / declaration
  }
  return out;
}

  10. 
        
    10
    Wire the UI: run on input, copy, show errors
    
        
    The whole engine is pure, so the UI layer is thin: on every keystroke, call the engine inside a try/catch. On success, drop the string into the output box and update the stats. On an XmlError, show a red banner with the message and the offending tag, and convert the byte position into a friendly line/column. Copy is one clipboard call.
    
        
    function run(){
  try {
    const result = mode === "minify" ? minifyXml(src) : formatXml(src, { indent });
    output.textContent = result; showStats(src, result); hideError();
  } catch (e){
    output.textContent = ""; showError(e);   // e.message + e.tag + line/col from e.pos
  }
}
input.addEventListener("input", run);
