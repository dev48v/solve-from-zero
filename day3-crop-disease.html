🔨 Build it for real

    
Estimated time: 2 hours. Stack: TensorFlow.js + PlantVillage dataset (54k images, 38 classes).

    1
    Get a pre-trained model
    
        
    PlantVillage = de-facto public dataset for crop disease (38 disease classes across 14 crops). MobileNetV2 transfer-learned on PlantVillage hits ~95% accuracy and is small enough to run in-browser via TF.js.
    

    # Convert any PyTorch/Keras model to TF.js layers format:
pip install tensorflowjs
tensorflowjs_converter --input_format=keras \
  plantvillage.h5 \
  ./model_tfjs/

    2
    Load + predict in-browser
    
        
    

    <script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs"></script>
<script>
const model = await tf.loadLayersModel("/model_tfjs/model.json");

// User picked a photo
fileInput.onchange = async (e) => {
  const img = new Image();
  img.src = URL.createObjectURL(e.target.files[0]);
  await img.decode();

  // Pre-process: resize to 224×224, normalize 0-1, add batch dim
  const tensor = tf.browser.fromPixels(img)
    .resizeBilinear([224, 224])
    .toFloat().div(255).expandDims();

  // Forward pass
  const probs = await model.predict(tensor).data();

  // Find top class
  const top = probs.indexOf(Math.max(...probs));
  console.log(CLASSES[top], "conf:", probs[top]);
};
</script>

    Offline-first PWA
    
        
    Rural villages have spotty 2G. Service worker caches model (~14 MB) on first install. App works fully offline after that. workbox-cli handles the cache config in one config file.

    Hindi / Marathi voice output
    
        
    Web Speech API speechSynthesis.speak with lang: "hi-IN" reads the remedy aloud — many farmers can't read but can listen. Bonus: pair with phone camera + voice = a complete diagnosis tool.
