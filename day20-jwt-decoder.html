🎫 JWT Decoder — Read Any Token (Safely)

Paste a JWT — read exactly what it says.

A JWT is just three base64url-encoded parts joined by dots — header.payload.signature. The header and payload are readable by anyone (so don't put secrets in them). The signature only proves the token wasn't tampered with if you verify it with the key. This decodes everything in your browser — nothing is uploaded, no library used.

This is the raw signature, shown as-is. Verifying it (proving the token is genuine) needs the secret or public key — that happens on the server, not here.
What this token says (standard claims, in plain English)
Standard claims are short three-letter keys defined by the JWT spec: iss issuer · sub subject · aud audience · exp expires · iat issued-at · nbf not-before. Times are seconds since 1970 (Unix time); we render them as your local date.

How the decoding works: we split the token on the two dots into three strings, then base64url-decode the first two. base64url is regular base64 with +→- and /→_ and the = padding stripped, so we reverse those swaps, re-pad to a multiple of 4, run atob, then UTF-8-decode the bytes and JSON.parse. The signature stays encoded because it's raw bytes, not text — and we never verify it, because that requires the signing key.

🔨 Build it yourself

    
Split the token on the dots → base64url-decode the first two parts → JSON.parse them → read the standard claims → turn the Unix timestamps into real dates → check expiry. ~30 lines for a correct, library-free decoder. One thing it deliberately does not do: verify the signature — that needs the signing key and belongs on the server.

    
    
      
  1. 1
    Split the token on the dots
    // a JWT is exactly three base64url strings joined by ".".
const token = "eyJhbGciOi…";
const [headerB64, payloadB64, signatureB64] = token.split(".");
// must be exactly 3 parts — anything else is malformed.
if (token.split(".").length !== 3) throw new Error("Not a JWT");
    2. 

      
  2. 2
    base64url → bytes → text
    // base64url ≠ plain base64: + → -, / → _, and no "=" padding.
// reverse those, re-pad to a multiple of 4, then atob().
function base64UrlDecode(str){
  str = str.replace(/-/g, "+").replace(/_/g, "/");
  while (str.length % 4) str += "=";          // restore padding
  const bytes = atob(str);                    // base64 → binary string
  // decode the bytes as UTF-8 so emoji / accents survive:
  return decodeURIComponent(
    [...bytes].map(c => "%" + c.charCodeAt(0).toString(16).padStart(2,"0")).join("")
  );
}

  3. 3
    Parse header & payload as JSON
    // the decoded strings are JSON — parse them into objects.
const header  = JSON.parse(base64UrlDecode(headerB64));
const payload = JSON.parse(base64UrlDecode(payloadB64));
// header  → { alg: "HS256", typ: "JWT" }
// payload → { sub: "1234567890", name: "Jane", iat: 1700000000, … }
// wrap in try/catch — bad base64 or bad JSON should error cleanly.
    4. 

      
  4. 4
    Read the standard claims
    // the spec defines short, well-known claim keys:
const meaning = {
  iss: "Issuer — who created the token",
  sub: "Subject — who the token is about (often a user id)",
  aud: "Audience — who the token is intended for",
  exp: "Expires at",
  iat: "Issued at",
  nbf: "Not valid before",
};
// everything else in the payload is a custom claim (name, role, …).
    5. 

      
  5. 5
    Unix seconds → real dates
    // exp / iat / nbf are SECONDS since 1970 (not milliseconds).
// JavaScript Date wants milliseconds, so multiply by 1000.
function toDate(unixSeconds){
  return new Date(unixSeconds * 1000).toLocaleString();
}
toDate(payload.iat);  // → "Sat, 14 Nov 2023, 5:13:20 PM"
toDate(payload.exp);  // → "Sat, 14 Nov 2023, 6:13:20 PM"

  6. 6
    Is it expired? (with clock skew)
    // compare exp against "now" in seconds.
const now = Math.floor(Date.now() / 1000);
const skew = 60;                              // allow 60s clock drift
const expired = payload.exp && now > payload.exp + skew;

if (expired){
  const mins = Math.round((now - payload.exp) / 60);
  console.log(`Expired ${mins} min ago`);
} else if (payload.exp){
  const mins = Math.round((payload.exp - now) / 60);
  console.log(`Expires in ${mins} min`);
}
    7. 

      
  7. 7
    Decoding ≠ verifying (important)
    // what we did above only DECODES — it reads the claims.
// it does NOT prove the token is genuine. Anyone can craft
// a fake token with any payload they like.
//
// VERIFYING checks the signature against the signing key:
//   HS256 → recompute HMAC(header.payload, secret), compare
//   RS256 → verify the RSA signature with the public key
//
// → always verify on the SERVER. never trust a decoded
//   payload in the browser as proof of anything.
    8. 

      
  8. 8
    Decode in one function
    function decodeJwt(token){
  const parts = token.split(".");
  if (parts.length !== 3) throw new Error("A JWT has 3 parts");
  const [h, p, sig] = parts;
  return {
    header:    JSON.parse(base64UrlDecode(h)),
    payload:   JSON.parse(base64UrlDecode(p)),
    signature: sig,            // left encoded — it's raw bytes
    verified:  false,          // decoding never verifies
  };
}
decodeJwt(token);
// ➜ { header:{alg:"HS256",typ:"JWT"}, payload:{…}, signature:"…", verified:false }
