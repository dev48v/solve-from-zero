🔀 Diff Checker — Spot Every Change With LCS

Compare two texts, see every add and delete — all in your browser

Nothing is uploaded. Both texts are split into lines, a Longest Common Subsequence is computed, and anything not on that shared backbone is shown as a green addition or a red deletion.

🔨 Build it yourself

    
Split both texts into lines, build the LCS dynamic-programming table, backtrack it into keep/add/remove operations, then render them. ~45 lines for a real line diff.

    
    
      
  1. 1
    Split both texts into arrays of lines
    const A = original.split("\n");   // old lines
const B = changed.split("\n");    // new lines
// a diff compares A vs B element-by-element, where each
// "element" is one whole line of text.
    2. 

      
  2. 2
    Build the LCS length table (dynamic programming)
    // dp[i][j] = length of the LCS of A[0..i-1] and B[0..j-1]
const n = A.length, m = B.length;
const dp = Array.from({length:n+1}, () => new Array(m+1).fill(0));
for (let i = 1; i <= n; i++) {
  for (let j = 1; j <= m; j++) {
    dp[i][j] = A[i-1] === B[j-1]
      ? dp[i-1][j-1] + 1                  // lines match → extend LCS
      : Math.max(dp[i-1][j], dp[i][j-1]); // else best of skipping one
  }
}
    3. 

      
  3. 3
    Backtrack the table into operations
    // walk from the bottom-right corner back to (0,0)
const ops = [];
let i = n, j = m;
while (i > 0 && j > 0) {
  if (A[i-1] === B[j-1]) {                 // on the LCS → unchanged
    ops.push({type:"keep", text:A[i-1]}); i--; j--;
  } else if (dp[i-1][j] >= dp[i][j-1]) {   // came from above → removed
    ops.push({type:"del",  text:A[i-1]}); i--;
  } else {                                 // came from left → added
    ops.push({type:"add",  text:B[j-1]}); j--;
  }
}
    4. 

      
  4. 4
    Drain the leftovers, then reverse
    // one side may still have lines once the other ran out
while (i > 0) { ops.push({type:"del", text:A[--i]}); }
while (j > 0) { ops.push({type:"add", text:B[--j]}); }
ops.reverse();   // we built it backwards, so flip to top-down order
    5. 

      
  5. 5
    Count the stats
    const added   = ops.filter(o => o.type === "add").length;
const removed = ops.filter(o => o.type === "del").length;
const kept    = ops.filter(o => o.type === "keep").length;
// identical inputs → added === 0 && removed === 0
    6. 

      
  6. 6
    Render a unified view
    const sign = { keep:" ", add:"+", del:"-" };
const out = ops.map(o => {
  const cls = o.type === "add" ? "diff-add"
            : o.type === "del" ? "diff-del" : "diff-keep";
  return `<div class="diff-row ${cls}">`
       + `<span class="diff-gutter">${sign[o.type]}</span>`
       + `<span>${escapeHtml(o.text)}</span></div>`;
}).join("");
diffEl.innerHTML = out;
    7. 

      
  7. 7
    Live update, debounced
    let t;
function schedule() { clearTimeout(t); t = setTimeout(render, 150); }
left.addEventListener("input", schedule);
right.addEventListener("input", schedule);
// debouncing waits 150ms after the last keystroke so we don't
// recompute the whole table on every single character.
    8. 

      
  8. 8
    Optional: word-level diff inside a changed line
    // run the exact same LCS, but on words instead of lines
function wordDiff(oldLine, newLine) {
  const a = oldLine.split(/(\s+)/), b = newLine.split(/(\s+)/);
  const ops = lcsDiff(a, b);   // reuse steps 2–4!
  return ops;                  // then wrap add/del spans inline
}
// LCS doesn't care what the "elements" are — lines, words, or chars.
