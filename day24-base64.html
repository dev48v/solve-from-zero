🔤 Base64 Encoder / Decoder — text ⇄ Base64, UTF-8 safe

Type text on the left, Base64 on the right — either side updates the other, live.

Base64 turns any bytes into a string of 64 plain characters (A–Z a–z 0–9 + /) so binary can travel safely through channels that only tolerate text — email bodies, JSON, URLs, data-URIs. It works by regrouping bits: every 3 bytes (24 bits) become 4 characters of 6 bits each. This tool does it the UTF-8-correct way (naïve btoa throws on é or 世), lets you flip to the URL-safe alphabet, visualises the bit-regrouping, and will Base64 any file you drop. All offline, no libraries.

🔨 Build it yourself

    
Encode text to Base64 the correct way: turn the string into UTF-8 bytes first, then regroup those bytes 3-at-a-time into 4 six-bit indices, and finally map each index through the alphabet. Decoding runs the pipeline in reverse. Wire two textareas so editing either re-renders the other, add a URL-safe toggle, and you have a real, UTF-8-safe tool in ~80 lines of vanilla JS.

    
    
      
  1. 
        
    1
    Text → UTF-8 bytes (do this first!)
    
        
    Base64 encodes bytes, not characters. A char like 世 is several bytes in UTF-8, and raw btoa("世") throws because it assumes each char fits in one byte. TextEncoder gives you the correct byte array for any Unicode string, so everything downstream just works.
    
        
    function toBytes(str){
  return new TextEncoder().encode(str);   // Uint8Array of UTF-8 bytes
}
toBytes("héllo 世界");
// [104,233?…] → é and 世 expand to 2–3 bytes each
    
      
    2. 
      
  2. 
        
    2
    The 64-character alphabet
    
        
    Standard Base64 indexes 0–63 into A–Z, then a–z, then 0–9, then + and /. That's exactly 64 symbols — the largest power-of-two alphabet made only of printable, widely-safe ASCII. Build it once as a string and look up by index.
    
        
    const STD = "ABCDEFGHIJKLMNOPQRSTUVWXYZ"
          + "abcdefghijklmnopqrstuvwxyz"
          + "0123456789+/";
STD[0];   // "A"
STD[63];  // "/"
STD.length; // 64
    
      
    3. 
      
  3. 
        
    3
    Regroup 3 bytes (24 bits) → 4 indices (6 bits)
    
        
    Take three bytes at a time. Glue their 24 bits together into one number, then slice off four 6-bit chunks (>> and & 63). Each chunk is 0–63 — a direct index into the alphabet. Three bytes in, four characters out: that's the whole engine.
    
        
    // bytes b0,b1,b2 →
const n = (b0 << 16) | (b1 << 8) | b2;   // 24-bit number
const c0 = (n >> 18) & 63;
const c1 = (n >> 12) & 63;
const c2 = (n >>  6) & 63;
const c3 =  n         & 63;
// out = STD[c0]+STD[c1]+STD[c2]+STD[c3]
    
      
    4. 
      
  4. 
        
    4
    Handle the leftover 1 or 2 bytes with =
    
        
    Input length isn't always a multiple of 3. With 1 leftover byte you still emit 2 chars (12 bits, zero-padded) then ==; with 2 leftover bytes you emit 3 chars then =. The = is not data — it tells the decoder how many real bytes the final group carried.
    
        
    // 1 leftover byte  → "XY=="   (2 chars + 2 pad)
// 2 leftover bytes → "XYZ="   (3 chars + 1 pad)
const pad = "=".repeat((3 - (len % 3)) % 3);
btoa-style: fill missing bytes with 0, then replace
the surplus output chars with "=".
    
      
    5. 
      
  5. 
        
    5
    Encode via btoa — but the UTF-8 way
    
        
    The browser's btoa is fast but only accepts a "binary string" (one byte per char). Feed it Unicode directly and it throws. The fix: TextEncoder → bytes → build a binary string byte-by-byte → btoa. Decoding mirrors it with atob → bytes → TextDecoder.
    
        
    function encode(str){
  const bytes = new TextEncoder().encode(str);
  let bin = "";
  for (const b of bytes) bin += String.fromCharCode(b);
  return btoa(bin);
}
function decode(b64){
  const bin = atob(b64);
  const bytes = Uint8Array.from(bin, c => c.charCodeAt(0));
  return new TextDecoder().decode(bytes);
}
    
      
    6. 
      
  6. 
        
    6
    URL-safe variant
    
        
    + and / mean things in URLs (space, path separator), so RFC 4648 defines a URL-safe alphabet swapping them for - and _. Padding = is often stripped too since %3D is ugly. Convert with two replaces after encoding, and reverse them before decoding.
    
        
    function toUrlSafe(b64){
  return b64.replace(/\+/g, "-").replace(/\//g, "_")
            .replace(/=+$/, "");        // drop padding
}
function fromUrlSafe(s){
  s = s.replace(/-/g, "+").replace(/_/g, "/");
  while (s.length % 4) s += "=";        // restore padding
  return s;
}

  7. 
        
    7
    File → Base64 data-URI
    
        
    FileReader.readAsDataURL hands you a ready-made data:<mime>;base64,… string — the exact form you can paste into an <img src> or CSS url() to embed an asset with no separate request. Great for tiny icons; wasteful for big files because of the ~33% overhead (next step).
    
        
    fileInput.onchange = e => {
  const file = e.target.files[0];
  const reader = new FileReader();
  reader.onload = () => console.log(reader.result);
  // reader.result = "data:image/png;base64,iVBORw0KG…"
  reader.readAsDataURL(file);
};

  8. 
        
    8
    Wire the two fields (and know it's not encryption)
    
        
    Editing text re-encodes into the Base64 box; editing Base64 decodes back into text — guard the flag so you don't re-trigger the field being typed in. Show the ~33% size overhead. And remember: Base64 is reversible by anyone — it hides nothing. Never treat it as a password store or a security layer.
    
        
    txtIn.oninput = () => { b64In.value = encode(txtIn.value); render(); };
b64In.oninput = () => {
  try { txtIn.value = decode(fromUrlSafe(b64In.value)); }
  catch { showError("Not valid Base64"); }
};
// overhead ≈ ceil(bytes/3)*4 / bytes  → ~1.33×
// ⚠ Base64 is encoding, NOT encryption.
