🌑 Box-Shadow Generator — a shadow is just four lengths and a colour; stack a few and you have depth

Drag the sliders, stack the layers, copy the exact box-shadow.

A CSS shadow is not a picture — it is a tiny recipe of four numbers and a colour: how far to nudge it right and down (offset-x / offset-y), how soft the edge is (blur), how much bigger or smaller than the box it is (spread), and the colour — whose alpha does most of the work. One line reads flat; the trick real designers use is to stack several faint shadows into one comma-separated list, so a card gets both a tight contact shadow and a wide ambient one. This tool assembles that list by hand and shows the exact value you paste into CSS. No library, all vanilla JS.

Five things that surprise people

        
1. The colour's alpha is the whole trick. A solid black shadow looks like a photocopier smudge; real shadows are rgba(0,0,0,0.1)-ish. Almost every "why does my shadow look cheap" problem is an opacity that is too high. 2. Blur and spread are different. Blur softens the edge outward from the shape; spread grows (positive) or shrinks (negative) the shadow's size before it is blurred — a negative spread is the secret to a tight, believable drop. 3. The order of the list is a stacking order. In a comma-separated box-shadow, the first shadow paints on top of the later ones — exactly the reverse of what people expect. 4. inset flips the light around. The same four numbers, with inset, carve the shadow into the box instead of casting it outward — that is how you make pressed buttons and inner wells. 5. Shadows are free of the layout. Unlike a bigger border or an extra element, a shadow takes up no space and never pushes its neighbours around; it also follows the box's border-radius automatically.

🔨 Build it yourself

    
A shadow generator has no algorithm to speak of — the cleverness is entirely in the data model. Hold each shadow as a plain object of five numbers plus a colour and an inset flag; hold the whole design as an array of those objects. Everything else is two tiny functions: one that turns a single object into a box-shadow fragment, and one that joins the array with commas. The sliders write into the selected object, you re-serialise, and you drop the string straight onto element.style.boxShadow. There is no library; the entire engine is the handful of functions below.

    
    
      
  1. 
        
    1
    Model one shadow as a plain object
    
        
    The CSS syntax [inset] <x> <y> <blur> <spread> <color> maps one-to-one onto an object. Keep the four lengths as plain numbers (pixels), the colour as an {r,g,b} triple, the transparency as a separate opacity percentage, and inset as a boolean. Splitting colour from opacity is deliberate: it lets one slider drive the alpha without touching the hue.
    
        
    // one shadow = the box-shadow syntax as data
// box-shadow: [inset] offset-x offset-y blur spread color
const layer = {
  x: 0, y: 10,          // offset-x, offset-y  (px)
  blur: 25, spread: -5, // blur-radius, spread-radius (px)
  color: { r:15, g:23, b:42 },
  opacity: 18,          // % -> the colour's alpha
  inset: false
};
    
      
    2. 
      
  2. 
        
    2
    Build the colour: hex + opacity → rgba()
    
        
    The native colour input gives you a #rrggbb hex, but a shadow needs an alpha. So convert the hex to red/green/blue integers, divide the opacity percentage by 100, and assemble an rgba() string. This one function is why the opacity slider feels like the most important control on the panel.
    
        
    function hexToRgb(hex){
  hex = hex.replace("#", "");
  if (hex.length === 3) hex = hex.split("").map(c => c + c).join("");
  const n = parseInt(hex, 16);
  return { r:(n >> 16) & 255, g:(n >> 8) & 255, b:n & 255 };
}
function rgba(layer){
  const { r, g, b } = layer.color;
  return `rgba(${r}, ${g}, ${b}, ${layer.opacity / 100})`;
}
    
      
    3. 
      
  3. 
        
    3
    Serialise one shadow to a string
    
        
    Now turn one object into one valid box-shadow fragment. Push the parts into an array in the exact CSS order, prepend inset only when the flag is set, and join with spaces. Building an array and joining — rather than string-concatenating — keeps the optional inset keyword clean and the output free of stray spaces.
    
        
    function shadowToString(s){
  const parts = [];
  if (s.inset) parts.push("inset");
  parts.push(s.x + "px", s.y + "px", s.blur + "px", s.spread + "px", rgba(s));
  return parts.join(" ");
}
// { x:0, y:10, blur:25, spread:-5, opacity:18 }
//   -> "0px 10px 25px -5px rgba(15, 23, 42, 0.18)"
    
      
    4. 
      
  4. 
        
    4
    Join the layers with commas = the value
    
        
    A stacked shadow is nothing more than several fragments in one comma-separated list. Map the whole array through shadowToString and join with ", ". That single string is the complete box-shadow value — the same text you would type by hand — and it is what makes multiple layers a first-class feature rather than a hack.
    
        
    function joinShadows(list){
  return list.map(shadowToString).join(", ");
}
// two layers ->
// "0px 10px 25px -5px rgba(15,23,42,0.15),
//  0px 4px 6px -2px rgba(15,23,42,0.10)"
    
      
    5. 
      
  5. 
        
    5
    State = an array + a selected index
    
        
    All the UI mutates just two variables: the shadows array and which layer is currently selected. The sliders never touch the DOM's style directly — they edit shadows[selected] and call one render(). Keeping a single source of truth is what makes add / remove / reorder trivial later.
    
        
    let shadows = [
  { x:0, y:10, blur:25, spread:-5, color:{r:15,g:23,b:42}, opacity:15, inset:false },
  { x:0, y:4,  blur:6,  spread:-2, color:{r:15,g:23,b:42}, opacity:10, inset:false }
];
let selected = 0;              // which layer the editor edits
    
      
    6. 
      
  6. 
        
    6
    Sliders write into the selected layer
    
        
    Every range input maps to one field of the selected object. On input, coerce the value to a number, store it, and re-render — live, on every pixel of the drag. Because the model is plain data, wiring a control is a one-liner and the preview updates instantly with no diffing.
    
        
    rngBlur.addEventListener("input", () => {
  shadows[selected].blur = +rngBlur.value;   // "+" -> number
  render();
});
insetTog.addEventListener("change", () => {
  shadows[selected].inset = insetTog.checked;
  render();
});
    
      
    7. 
      
  7. 
        
    7
    render(): one string, straight onto the box
    
        
    The render is the payoff. Build the value once, assign it to the preview element's style.boxShadow — the browser does all the actual shadow maths — and mirror the same string into the read-only CSS output. There is no canvas and no drawing loop; you are just handing CSS to the engine that already knows how to paint it.
    
        
    function render(){
  const css = joinShadows(shadows);
  box.style.boxShadow = css;                 // the browser paints it
  cssOut.textContent  = "box-shadow: " + css + ";";
  renderLayerList();                         // refresh the strip
  syncEditor();                              // sliders show selected layer
}
    
      
    8. 
      
  8. 
        
    8
    Add, remove, and select layers
    
        
    Multiple shadows fall out of the array for free. "Add" pushes a fresh object and selects it; "remove" splices one out and clamps the selection; clicking a row sets selected. Guard against removing the last layer so the box is never left with an invalid empty value.
    
        
    function addLayer(){
  shadows.push({ x:0, y:8, blur:16, spread:0,
                 color:{r:15,g:23,b:42}, opacity:20, inset:false });
  selected = shadows.length - 1; render();
}
function removeLayer(i){
  if (shadows.length === 1) return;          // keep at least one
  shadows.splice(i, 1);
  selected = Math.min(selected, shadows.length - 1); render();
}
    
      
    9. 
      
  9. 
        
    9
    Layering for real depth (the design part)
    
        
    One shadow looks flat; believable elevation comes from stacking a tight, darker contact shadow over a wide, fainter ambient one — mimicking how a direct light and the sky both cast shadows. Remember the list is a stacking order: put the sharp one first so it sits on top. Presets are just canned arrays of this shape.
    
        
    // "layered depth" preset — sharp on top, soft underneath
const softCard = [
  { x:0, y:2,  blur:4,  spread:-1, opacity:12, color:INK },  // contact
  { x:0, y:12, blur:24, spread:-6, opacity:14, color:INK }   // ambient
];
// order matters: first fragment paints ON TOP of the later ones
    
      
    10. 
      
  10. 
        
    10
    Copy the CSS — and why not filter?
    
        
    Copy is one clipboard call on the already-built string. And a closing choice: why box-shadow rather than filter: drop-shadow()? box-shadow follows the box's rectangle and border-radius, takes no layout space, and accepts a stacked list — perfect for cards and buttons. drop-shadow traces the alpha silhouette (great for irregular PNGs) but does not stack the same way. Pick the tool that matches the shape you are shadowing.
    
        
    copyCss.onclick = () => {
  const css = "box-shadow: " + joinShadows(shadows) + ";";
  navigator.clipboard.writeText(css);
  copyCss.textContent = "✓ Copied!";
  setTimeout(() => copyCss.textContent = "Copy CSS", 1200);
};
