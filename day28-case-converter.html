🔤 Case Converter — camelCase ⇄ snake_case ⇄ kebab-case ⇄ … with a smart word-splitter

Retype nothing. Convert any identifier to every case.

Case is the naming convention a piece of text follows — camelCase for JS variables, snake_case for Python and SQL columns, kebab-case for URLs and CSS, CONSTANT_CASE for env vars, PascalCase for classes, Title Case for headings. Converting between them by hand is error-prone. This tool does it for real in your browser: the heart of it is a word-splitter that tokenizes any input — it breaks on spaces, underscores, hyphens, dots and slashes, on camelCase humps, and at letter→digit boundaries, while keeping acronyms whole (HTTPServer → HTTP + Server). Once the words are known, re-joining them into any target case is trivial. Paste one identifier or a whole multi-line list — every line is converted independently. All offline, no libraries.

Some conversions are lossy — and that surprises people

snake_case, kebab-case, camelCase and PascalCase round-trip cleanly: their word boundaries are unambiguous, so user_id → userId → user_id comes back identical. But casing normalizes acronyms — HTTPServer becomes Http Server in Title Case, and there is no way to know later that HTTP was capitalized, so you can't recover it. And Title Case or Sentence case that contain real multi-word phrases lose the boundary information the moment they hit a space-free case: "New York" → newYork can never split back into New+York reliably. Rule of thumb: convert freely between the programmer cases, but treat human-readable cases as a one-way exit.

🔨 Build it yourself

    
A case converter is a two-phase pipeline with no dependencies. Phase one is a single robust function that splits any identifier into a clean list of words. Phase two is a table of tiny join functions, one per target case. Nail the splitter and every case is a one-liner. Below is the whole thing in vanilla JS — the same code powering the LOOK tab.

    
    
      
  1. 
        
    1
    Split on the obvious delimiters
    
        
    The easy half of tokenizing: break the string wherever a run of non-alphanumeric characters appears. That single split covers snake_case, kebab-case, dot.case, path/case and plain spaces at once. Filter out the empty strings a leading or trailing delimiter leaves behind.
    
        
    function splitDelims(str){
  return str.split(/[^A-Za-z0-9]+/).filter(Boolean);
}
splitDelims("my-var_name");   // ["my","var","name"]
splitDelims("a.b/c d");       // ["a","b","c","d"]

  2. 
        
    2
    Split camelCase & PascalCase humps
    
        
    fooBar has no delimiter — the boundary is the hump, a lowercase (or digit) immediately followed by an uppercase letter. Don't try to split it directly; instead inject a space at every hump, then let the delimiter split from step 1 do the work. One regex, one capture-group swap.
    
        
    // insert a space between  aB  ->  a B
const withSpace = str.replace(/([a-z0-9])([A-Z])/g, "$1 $2");
"getUserName".replace(/([a-z0-9])([A-Z])/g, "$1 $2");
// -> "get User Name"

  3. 
        
    3
    Keep acronyms whole
    
        
    Step 2 breaks HTTPServer in the wrong place, because P→S isn't a hump (both uppercase). The fix: a run of capitals followed by an uppercase-then-lowercase pair is an acronym butting against a word. Split before that last capital — so HTTPServer becomes HTTP Server, not HTTPS erver.
    
        
    // ACRONYM + Word:  HTTPServer -> HTTP Server
const fixed = str.replace(/([A-Z]+)([A-Z][a-z])/g, "$1 $2");
"HTTPServerName".replace(/([A-Z]+)([A-Z][a-z])/g, "$1 $2");
// -> "HTTP ServerName"  (then step 2 finishes the rest)
    
      
    4. 
      
  4. 
        
    4
    Handle digit boundaries
    
        
    A digit usually starts a new token: version2 is version + 2. Inject a space at every letter→digit boundary. Note we don't split digit→letter, so tokens like 2xx, 3d or utf8's tail stay glued to their leading digit — matching how people actually read them.
    
        
    // letter -> digit boundary:  Error2xx -> Error 2xx
const d = str.replace(/([A-Za-z])([0-9])/g, "$1 $2");
"Error2xx".replace(/([A-Za-z])([0-9])/g, "$1 $2");
// -> "Error 2xx"   (digit->letter is left intact)
    
      
    5. 
      
  5. 
        
    5
    The whole splitWords()
    
        
    Chain the three inserts in order — acronym boundary is applied via the same lower/upper pass plus the capitals rule — then split on delimiters. The order matters: fix humps and acronyms and digits first (they only add spaces), then the delimiter split collapses everything. This one function is the entire hard part of the tool.
    
        
    function splitWords(str){
  if (str == null) return [];
  return String(str)
    .replace(/([a-z0-9])([A-Z])/g, "$1 $2")     // camel hump
    .replace(/([A-Z]+)([A-Z][a-z])/g, "$1 $2")  // acronym | Word
    .replace(/([A-Za-z])([0-9])/g, "$1 $2")     // letter | digit
    .split(/[^A-Za-z0-9]+/)                      // delimiters
    .filter(Boolean);
}
splitWords("HTTPServerError2xx");
// -> ["HTTP","Server","Error","2xx"]
    
      
    6. 
      
  6. 
        
    6
    Join rules — one per target case
    
        
    With a clean word list, each case is a two-decision recipe: how do I transform each word (lower / UPPER / Capitalize) and what separator joins them (nothing / _ / - / . / / / space). cap() is the workhorse — first letter up, rest down — which is also what normalizes an acronym like HTTP down to Http.
    
        
    const cap = w => w.charAt(0).toUpperCase() + w.slice(1).toLowerCase();

const CONVERTERS = {
  "camelCase":     ws => ws.map((w,i)=> i ? cap(w) : w.toLowerCase()).join(""),
  "PascalCase":    ws => ws.map(cap).join(""),
  "snake_case":    ws => ws.map(w => w.toLowerCase()).join("_"),
  "kebab-case":    ws => ws.map(w => w.toLowerCase()).join("-"),
  "CONSTANT_CASE": ws => ws.map(w => w.toUpperCase()).join("_"),
  "Title Case":    ws => ws.map(cap).join(" "),
  "Sentence case": ws => ws.map((w,i)=> i ? w.toLowerCase() : cap(w)).join(" "),
  "dot.case":      ws => ws.map(w => w.toLowerCase()).join("."),
  "path/case":     ws => ws.map(w => w.toLowerCase()).join("/"),
  "Train-Case":    ws => ws.map(cap).join("-"),
};

  7. 
        
    7
    Detect the input case
    
        
    Detection is a bonus, not a requirement — conversion never needs it, because the splitter is case-agnostic. But it's a nice touch. Test for the tell-tale marks in priority order: a separator character wins first, then the shape of the capitals. Whatever you can't classify is honestly reported as mixed / unknown.
    
        
    function detectCase(s){
  s = String(s).trim();
  if (!s) return "empty";
  if (/\s/.test(s))    return /^[A-Z][a-z0-9]*(\s[A-Z][a-z0-9]*)*$/.test(s)
                              ? "Title Case" : "Sentence case / words";
  if (s.includes("_")) return /[a-z]/.test(s) ? "snake_case" : "CONSTANT_CASE";
  if (s.includes("-")) return /-[A-Z]/.test(s) ? "Train-Case" : "kebab-case";
  if (s.includes("/")) return "path/case";
  if (s.includes(".")) return "dot.case";
  if (/^[a-z][A-Za-z0-9]*$/.test(s) && /[A-Z]/.test(s)) return "camelCase";
  if (/^[A-Z][A-Za-z0-9]*$/.test(s) && /[a-z]/.test(s)) return "PascalCase";
  return "mixed / unknown";
}

    8
    Wire it up — multi-line convert()
    
        
    The public function converts a whole block: split on newlines, run each line through splitWords then the chosen join, and rejoin with newlines. Because each line tokenizes independently, a mixed list — camel, snake and kebab all at once — normalizes perfectly into whatever case you pick.
    
        
    function convert(text, caseName){
  const join = CONVERTERS[caseName];
  return String(text).split("\n").map(line => {
    const ws = splitWords(line);
    return ws.length ? join(ws) : "";
  }).join("\n");
}
convert("user_id\ngetName\nAPI-key", "CONSTANT_CASE");
// -> "USER_ID\nGET_NAME\nAPI_KEY"
