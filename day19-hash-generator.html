🔐 Hash Generator — SHA in the Browser (Web Crypto)

Type anything — get its real cryptographic fingerprint.

This computes a genuine hash with the browser's built-in crypto.subtle.digest (the Web Crypto API). A hash is a one-way fingerprint: the same input always gives the same hash, any change gives a totally different one, and you can't run it backwards to recover the input. Nothing is uploaded — it all runs in your browser.

Your input
Reads the file as bytes and hashes them — large files take a moment.
🛈 MD5 isn't available in Web Crypto — it's cryptographically broken (cheap to forge collisions), so browsers deliberately leave it out.
Hash (hex)
…
SHA-256 256 bits 64 hex chars from text
A SHA-256 digest is always exactly 256 bits = 32 bytes = 64 hexadecimal characters, no matter whether you hashed one letter or a gigabyte.
The avalanche effect — change one character, get a completely different hash
Your input
One tiny tweak

How the fingerprint is built: for text we run it through a TextEncoder to get the raw UTF-8 bytes; for a file we read it as an ArrayBuffer of bytes. Either way we hand those bytes to crypto.subtle.digest(algorithm, bytes), which returns the digest as an ArrayBuffer. We turn each byte into a two-digit hex pair and join them — that's the string you see. Same bytes in → same hash out, every time, on every machine; flip a single bit and roughly half the output bits flip with it.

🔨 Build it yourself

    
Turn the input into bytes (TextEncoder for strings, ArrayBuffer for files) → hand the bytes to crypto.subtle.digest → convert the returned ArrayBuffer to a hex string → let the user pick the algorithm. ~25 lines for a correct, library-free hasher — the cryptography is the browser's, you only do the plumbing.

    
    
      
  1. 1
    String → bytes (TextEncoder)
    // crypto works on bytes, not strings.
// TextEncoder gives you the raw UTF-8 bytes of any text.
const bytes = new TextEncoder().encode("abc");
// → Uint8Array(3) [ 97, 98, 99 ]   ("a"=97, "b"=98, "c"=99)
    2. 

      
  2. 2
    Hash the bytes (crypto.subtle.digest)
    // the actual hashing. it's async and returns a Promise
// that resolves to an ArrayBuffer of raw digest bytes.
const digest = await crypto.subtle.digest("SHA-256", bytes);
// digest is an ArrayBuffer — 32 bytes for SHA-256.
    3. 

      
  3. 3
    ArrayBuffer → hex string
    // the digest is raw bytes; turn each into 2 hex digits.
function toHex(buffer){
  return [...new Uint8Array(buffer)]
    .map(b => b.toString(16).padStart(2, "0"))
    .join("");
}
// → "ba7816bf8f01cfea414140de5dae2223..."
    4. 

      
  4. 4
    Hash text in one function
    // put it together: text in, hex hash out.
async function hashText(text, algo = "SHA-256"){
  const bytes  = new TextEncoder().encode(text);
  const digest = await crypto.subtle.digest(algo, bytes);
  return toHex(digest);
}
await hashText("abc");
// ➜ "ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad"
    5. 

      
  5. 5
    Switch algorithms
    // Web Crypto supports the SHA-2 family (+ legacy SHA-1).
// the length of the output is fixed by the algorithm:
const ALGOS = {
  "SHA-1":   160,  // 40 hex chars — BROKEN, don't use for security
  "SHA-256": 256,  // 64 hex chars — the default everyone reaches for
  "SHA-384": 384,  // 96 hex chars
  "SHA-512": 512,  // 128 hex chars
};
// note: MD5 is NOT here — it's broken and Web Crypto omits it.
    6. 

      
  6. 6
    Hash a file (ArrayBuffer)
    // files are already bytes — read them as an ArrayBuffer
// and hand that straight to digest(). no TextEncoder needed.
async function hashFile(file, algo = "SHA-256"){
  const buffer = await file.arrayBuffer();   // reads the whole file
  const digest = await crypto.subtle.digest(algo, buffer);
  return toHex(digest);
}
// this is exactly how a "checksum" / download-verify works.
    7. 

      
  7. 7
    See the avalanche effect
    // change ONE character → roughly half the bits flip.
await hashText("hello");
// ➜ 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
await hashText("hellp");   // last letter o → p
// ➜ fdd7585e08c4e2afd71dcabdb4636c89d557a3f42db9e2040c8bbd1708aa4ce7
// totally different — that's what makes tampering obvious.

  8. 8
    Wire it to the page
    async function update(){
  const algo = algoSelect.value;
  const hex  = await hashText(textInput.value, algo);
  hashEl.textContent = hex;
  bitsEl.textContent = (hex.length * 4) + " bits";   // 4 bits / hex char
}
textInput.addEventListener("input", debounce(update, 150));
algoSelect.addEventListener("change", update);
update();   // run once on load — done.

A hash is a one-way fingerprint. Bytes in via TextEncoder or ArrayBuffer, crypto.subtle.digest does the real math, hex out. Use SHA-256 for integrity — never plain SHA for passwords (use salted bcrypt/argon2), and never MD5/SHA-1 for security.
