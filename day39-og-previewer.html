🔗 Meta-Tag / OG Previewer — one set of tags in your <head>, three very different social cards

Fill the fields or paste a <head> — watch the Google, X and Facebook/LinkedIn cards, then copy the tags.

When someone shares your link, the platform's crawler fetches the page and reads a handful of <meta> tags out of the <head> to build a preview card. There are two overlapping vocabularies: Open Graph (og:*, used by Facebook, LinkedIn, Slack, WhatsApp and most others) and Twitter Cards (twitter:*, used by X). Each platform reads what it wants, falls back when a tag is missing (og:title ← <title>, twitter:* ← og:*), and truncates differently — Google clips the title near 60 chars, X and Facebook clip at other lengths. The image wants a 1200×630 (1.91:1) ratio or it gets cropped. This tool resolves that whole fallback chain live, shows the three real-world cards, and writes the exact tag block for you to paste. No library — just string work and the browser's own layout.

Five things that surprise people

        
1. og: uses property=, twitter: uses name=. <meta property="og:title"> but <meta name="twitter:card">. Swap the attribute and the tag is silently ignored — the single most common "why is my card blank?" bug. 2. Missing tags fall back, they don't fail. No og:title? Facebook uses the <title>. No twitter:image? X uses og:image. You rarely need to repeat yourself — write the Open Graph set well and Twitter mostly inherits it. 3. Every platform truncates at a different length. Google clips the title around 60 chars and the description around 160; X and Facebook clip elsewhere. There is no single "correct" length — write for the shortest cutoff you care about. 4. The image must be an ABSOLUTE URL and roughly 1200×630. Relative paths and wrong ratios are why images vanish or get cropped; large cards want 1.91:1, and the file should be reachable by an anonymous crawler (no login wall). 5. Crawlers don't run your JavaScript and they CACHE. Tags injected by client-side JS usually won't be seen — they must be in the HTML the server returns. And once a platform scrapes your URL it remembers; use each platform's debugger to force a re-scrape after you change tags.

🔨 Build it yourself

    
A meta-tag previewer has one clever idea and a lot of easy plumbing. The clever idea: the whole tool is a fallback resolver — a tiny function that, for each platform, walks a list of tags and takes the first non-empty one (twitter:title → og:title → <title>). Everything else is data: six base fields in, a resolved og{}/twitter{} object out, three renderers that drop those strings into HTML, and a serialiser that prints the tag block. The only real-world concept you must keep straight is who reads what and how far it truncates. There is no library.

    
    
      
  1. 
        
    1
    The whole state is six base fields
    
        
    You do not store a tag per platform — you store the source of truth once and derive every tag from it. Title, description, URL, image, site name, and the Twitter card type. That flat object is your document; the previews are just views of it.
    
        
    let state = {
  title:       "How to Build a Meta-Tag Previewer",
  description: "A hands-on guide to Open Graph & Twitter Cards…",
  url:         "https://solvefromzero.dev/blog/og-previewer",
  image:       "https://solvefromzero.dev/img/og-cover.png",
  siteName:    "SolveFromZero",
  card:        "summary_large_image"   // or "summary"
};

    2
    The fallback rule is "first non-empty wins"
    
        
    This one helper is the heart of the whole tool. Give it a list of candidates in priority order and it returns the first that actually has a value. Every fallback chain in Open Graph and Twitter Cards is just a call to this.
    
        
    function pick(...vals){
  for (const v of vals)
    if (v != null && String(v).trim() !== "") return String(v);
  return "";
}
// pick(state.twTitle, state.ogTitle, state.title)  -> first one that exists

    3
    Resolve the base fields into og{} and twitter{}
    
        
    Now wire the chains. Open Graph reads the base fields; Twitter reads its own tag, then Open Graph, then the base — so a page that only sets og:* still gets a full X card. This function encodes the entire cross-platform fallback model in a dozen lines.
    
        
    function resolve(s){
  const title = pick(s.title), desc = pick(s.description);
  const url = pick(s.url), image = pick(s.image);
  return {
    docTitle: title, metaDesc: desc, canonical: url,
    og: { type:"website", title, description:desc, url, image, siteName:pick(s.siteName) },
    tw: {
      card:  pick(s.card, "summary_large_image"),
      title: pick(s.twTitle, title),       // twitter:title ← og:title ← <title>
      description: pick(s.twDescription, desc),
      image: pick(s.twImage, image)        // twitter:image ← og:image
    }
  };
}

    4
    Truncation is one function + a table
    
        
    Each surface clips text at a different length, so keep the limits as data and share one truncate. Cut at the limit, trim trailing space, add an ellipsis — that is the entire "this is what Google will actually show" logic.
    
        
    const LIMITS = {
  google:  { title: 60, desc: 160 },
  twitter: { title: 70, desc: 200 },
  facebook:{ title: 88, desc: 200 }
};
function truncate(str, n){
  str = str || "";
  return str.length <= n ? str : str.slice(0, n - 1).trimEnd() + "…";
}

  5. 
        
    5
    Parse a pasted <head> (no DOM needed)
    
        
    To read an existing page, sweep every <meta> tag and key it by its property or name. A small attribute-reader plus a regex loop is enough — and it runs anywhere, even under Node, because it never touches the DOM. (In the browser you could equally use DOMParser.)
    
        
    function attr(tag, name){
  const m = tag.match(new RegExp(name + '\\s*=\\s*"([^"]*)"', 'i'));
  return m ? m[1] : null;
}
function parseHead(html){
  const out = {};
  const t = html.match(/<title[^>]*>([\s\S]*?)<\/title>/i);
  if (t) out.title = t[1].trim();
  for (const m of html.matchAll(/<meta\b[^>]*>/gi)){
    const key = (attr(m[0],"property") || attr(m[0],"name") || "").toLowerCase();
    const content = attr(m[0], "content");
    if (key === "description")   out.description = content;
    if (key === "og:image")      out.image = content;   // …and the rest
  }
  return out;
}

    Render #1 — the Google result
    
        
    A search result is a favicon + site name, the URL as a breadcrumb, a blue title clipped at 60, and a grey description clipped at 160. Feed the resolved values through truncate and drop them into markup — the preview is just HTML.
    
        
    function serpHtml(t){
  const dom = domainOf(t.canonical);
  return `<div class="serp">
    <div class="serp-site">${t.og.siteName || dom}</div>
    <div class="serp-url">${breadcrumb(t.canonical)}</div>
    <div class="serp-title">${truncate(t.docTitle, LIMITS.google.title)}</div>
    <div class="serp-desc">${truncate(t.metaDesc, LIMITS.google.desc)}</div>
  </div>`;
}

    7
    Render #2 — the X card (two shapes)
    
        
    The Twitter card type decides the layout: summary_large_image puts a big 1.91:1 image on top, summary puts a small square thumbnail on the left. Same data, one if. The image box shows the picture when it loads and a placeholder when it can't — so an unreachable image never breaks the card.
    
        
    function twHtml(t){
  const large = t.tw.card !== "summary";
  const meta = `<div class="twdomain">${domainOf(t.canonical)}</div>
    <div class="twtitle">${truncate(t.tw.title, LIMITS.twitter.title)}</div>
    <div class="twdesc">${truncate(t.tw.description, LIMITS.twitter.desc)}</div>`;
  return large
    ? `<div class="twcard">${imgBox(t.tw.image)}<div class="twmeta">${meta}</div></div>`
    : `<div class="twcard small">${imgBox(t.tw.image)}<div class="twmeta">${meta}</div></div>`;
}

    Render #3 — Facebook / LinkedIn (Open Graph)
    
        
    Facebook and LinkedIn read the same Open Graph tags, so they share one renderer: image on top, then a grey bar with the domain in UPPERCASE, a bold title, and the description. This is why getting og:* right covers most of the internet at once.
    
        
    function fbHtml(t){
  return `<div class="fbcard">
    ${imgBox(t.og.image)}
    <div class="fbmeta">
      <div class="fbdomain">${domainOf(t.og.url).toUpperCase()}</div>
      <div class="fbtitle">${truncate(t.og.title, LIMITS.facebook.title)}</div>
      <div class="fbdesc">${truncate(t.og.description, LIMITS.facebook.desc)}</div>
    </div>
  </div>`;
}

    9
    Serialise the tag block — the deliverable
    
        
    The export is the resolved values printed as tags — remembering the attribute gotcha: og:* is property=, twitter:* is name=. Escape the content, join with newlines, hand it to the clipboard. That single string is the whole point of the tool.
    
        
    function buildTags(s){
  const t = resolve(s), e = escAttr;
  return [
    `<title>${e(t.docTitle)}</title>`,
    `<meta name="description" content="${e(t.metaDesc)}">`,
    `<meta property="og:title" content="${e(t.og.title)}">`,
    `<meta property="og:image" content="${e(t.og.image)}">`,
    `<meta name="twitter:card" content="${e(t.tw.card)}">`,
    `<meta name="twitter:image" content="${e(t.tw.image)}">`
    // …og:type/description/url/site_name + twitter:title/description
  ].join("\n");
}
copyBtn.onclick = () => navigator.clipboard.writeText(buildTags(state));

    10
    Warnings · check the image with the browser
    
        
    Lengths you can check against the LIMITS table. For the image ratio, don't compute anything — let the browser load it and read naturalWidth/naturalHeight. A token guards against a stale image finishing after the field changed. That is the whole tool: resolve, render, warn, copy.
    
        
    let imgToken = 0;
function checkImage(src){
  const token = ++imgToken;
  const im = new Image();
  im.onload = () => {
    if (token !== imgToken) return;            // a newer check started — ignore
    const w = im.naturalWidth, h = im.naturalHeight;
    const ok = Math.abs(w / h - 1.905) < 0.06; // ~1.91:1
    warnImage(ok, `${w}×${h}`);
  };
  im.onerror = () => warnImage(null, "couldn't load (fine once served)");
  im.src = src;
}
